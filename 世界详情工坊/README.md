# 世界详情工坊 · 工作流

> 目标：为世界库全部世界（主库 1346 + 休闲 1184 ≈ 2530 个）各产出**两个世界书条目**：
> `<世界名>·剧情`（≥10000 字·常青档案，卡片生成 + 入世后正文都读）与 `<世界名>·阶位切入点`（逐阶开场剧本，仅世界选择时读）。
> **铁则：一次只做一个世界。禁止「一批 5 个一起生成」。** 金标准样板见 [样板/我欲封天.md](样板/我欲封天.md)。

## 0. 每次动手前必读（缺一不可）

**每开始一个新世界之前**，执行者（Claude / 实惠模型 / 你）必须按顺序做完下面四步，**禁止跳步、禁止凭记忆开写、禁止并行开多个世界**：

| 步 | 动作 | 完成标准 |
|---|---|---|
| ① | **通读本 README 全文** + `模板/剧情条目模板.md` + `模板/切入点条目模板.md` + `模板/工单模板.md` 铁令 | 能复述：真名/禁灌水/字数/阶位覆盖/顶点规则 |
| ② | 从 `清单/批次表.md` 或用户点名取**仅 1 个**世界名；查 `清单/manifest.json` 得 `lib` + `tiers` | 输出路径、阶位覆盖已写清 |
| ③ | **联网检索免费公开资料**，至少读完「完整剧情脉络」（分卷/阶段 + 结局方向），并打开 ≥3 个可溯源 https 页 | 笔记里有：分卷因果、真名人物表、境界表、宝物真名、名场面 |
| ④ | 对照 `参考/阶位战力图鉴.md` 草拟乐园阶位映射（宁低勿高） | 映射行可写入正文 |

未完成 ①～④ → **不许写 `产出/` 下任何 md**。  
一个世界机检+QA 过关后，才允许取下一个世界。

## 内容质量铁则（写 / 改 / QA 一律遵守）

> 批次01 金标准见 `产出/批次01/`；批次02 真名范例见 `产出/批次02/`（莽荒纪/星辰变/修真四万年/超神机械师/诡秘之主）。

### A. 人物必须真名
- **【主要人物】与切入点「关键NPC立场」一律写原作真名**（如：纪宁、秦羽、李耀、韩萧、克莱恩·莫雷蒂）。
- **禁止代称/笼统标签**当人名用，包括但不限于：
  - `群像` / `众人模板` / `路人强者群像`
  - `红颜` / `核心道侣线` / `女主线之一`（须写成具体姓名）
  - `牙人` / `眼线` / `气压` / `标尺角色` / `相关存在` / `代行者`（可作身份后缀，**前面必须有真名**）
  - `核心同伴/红颜（原作名）` 这类占位句——**搜不到真名就写「不详」，不许用占位符充数**
- 每人固定字段：`姓名（身份）｜性格｜装备·能力｜人物弧光｜立场关系`；主角写全，配角也要真名。

### B. 禁止重复内容 / 禁止凑字
- **同一文件内禁止大段复制粘贴**（机检/QA 发现重复段 = 返工）。
- **禁止**下列凑字标记与段落（出现即返工）：
  - `【扩写·…】` / `【补密…】` / `【阶段档案…】` / `【剧情补述N】` / `【可介入事件·清单N】` / `【细目N·…】` 循环灌水
  - **`【加厚·…】` / `【加厚·档案】` / `【加厚·二轮】` / `【加厚·三轮·…】` / `【细则·…】` / `【补段N】` / `【扩段N】` / `【再补…】` / `【终卷补强…】` / `【叙事执行细则】` 等「为凑字数后贴」的元写作段**（不是原作剧情，禁止进成品）
  - 同一观点换皮重复（先写一句金句，再 `细则1…20` 每条复读 +「应转化为可观察细节」套话）
  - `跨媒介流行作品` / `可被契约者切入的完整任务世界` 等**跨世界通用套话**
  - `本阶可刷：假货、护送…` 这类**各阶复制同一句**的切入点填充
  - 仅改编号的重复段（清单14/15/16…、细目循环粘贴）
- **正文只保留模板规定的【】段**（作品来源…叙事基调·雷区）+ 切入点七字段 + 来源。需要加字 → 只能加**原作分卷事实、真名、宝物来历、名场面因果**，禁止加「写作指南式」段落。
- 字数不够时：**禁止**用空话、同义反复、跨世界模板、加厚轮次堆到 10000。正确做法见下条 C。

### C. 检索优先 · 完整剧情 · 唯一合法加字法
1. **写之前必须联网搜**，优先免费公开页：
   - 百科：百度百科 / 萌娘百科 / Fandom wiki / 维基
   - 作品页：起点作品页、出版方介绍
   - **完整剧情 / 原版全本核对（强制）**：
     - 主站：**[搜笔趣阁](https://www.sobqg.com/)**
     - **站内搜索（唯一正确入口）**：  
       `https://www.sobqg.com/searchBook.html?keyword=<书名URL编码>`  
       例：莽荒纪 → `https://www.sobqg.com/searchBook.html?keyword=%E8%8E%BD%E8%8D%92%E7%BA%AA`  
       （`method=get`，参数名必须是 **`keyword`**，不要用 `searchkey` 或瞎猜 `/book/拼音.html`）
     - 从搜索结果点进书页 `/book/<slug>.html`（slug 可能是拼音，也可能是 **`s2-` 前缀**，如莽荒纪=`/book/s2-manghuangji.html`；**禁止**仅因 `manghuangji` 404 就写「未收录」）
     - 书页确认作者/完结/字数 → **免费阅读**分卷关键章与终章 → 再写 md
     - 辅源：百科、起点作品页、高质量分卷长评（须实际打开）
   - 查不到写「不详」；**禁止编造**；成品不得整章粘贴原文（只允许总结进模板【】段）
2. 至少补搜并写入：**分卷/完整主线剧情**、**具名人物表**、**境界/等级表**、**关键宝物真名**、**名场面因果**、**结局方向**。
3. 用检索结果**扩写本世界独有的**剧情线/人物/宝物/地理，直到 `## 剧情` 去空白 ≥10000、切入点 ≥1500。
4. `## 来源` 必须挂上**你实际打开过的** ≥3 个 https 链接（禁止假链接）；其中**至少 1 条为搜笔趣阁该书页**（优先）或章节页；搜索页 URL 可作辅助，不能单独顶替书页。

### D. 格式与乐园侧
- 段落标题与工单模板一字不差；`# 世界名` 与清单全名精确一致。
- 乐园侧用语：阶位 / 契约者；「乐园阶位映射」与切入点「阶位↔」对照《阶位战力图鉴》，宁低勿高。
- 顶点：**超阶巅峰 = 存在·情报优先 / 条件性胜利**；严禁「被封印/被削弱所以战力为零」。

### E. 禁止事项（脚本与节奏）
- `scripts/auto-batch.mjs` **已停用**（通用模板生成，产出不合格）。
- 允许使用的脚本：`compile-worldbook.mjs`（机检/编译）、`gen-manifest.mjs`、`sync-tier-codex.mjs`。
- **禁止**「一次生成整批 5 世界」的自动化；允许的最小单位 = **1 个世界 md**。
- 新写工具前必须保证：**每世界独立检索字段**，不得输出跨世界相同正文。

## 条目规格（编译器自动生成，写手无需关心）

- 每世界 2 条：`<世界名>·剧情`（key=[世界名]）与 `<世界名>·阶位切入点`（key=[世界名]，keysecondary=[切入点]）
- 均为绿灯（selective=true，constant=false），enabled=true
- comment 后缀 `·剧情` / `·阶位切入点` 是程序识别标志，**后续前端按它区分注入场景**

## 字数配额（剧情条目 ≥10000 字，剧情优先）

| 段落 | 配额 | 要点 |
|---|---|---|
| 作品来源 | ~200 | 作者/媒介/系列/文风 |
| 世界定位 | ~150 | 一句话锚定 |
| 世界观·力量体系 | ~1200 | 境界/等级**逐级**列全 + 各级战力描述 + 乐园阶位映射 |
| 地理·舞台 | ~600 | 地图层级、主要地点 |
| **世界剧情线** | **~4000** | 分卷/分阶段**详述完整主线**，含关键转折、名场面、因果链、结局方向（大头） |
| **主要人物** | **~2500** | ≥10 人；**必须原作真名**；每人=身份/**性格**/**装备·能力**/人物弧光/立场关系 |
| 势力图谱 | ~600 | 各势力宗旨、地盘、**具名**代表人物、互相关系 |
| 贵重物品 | ~600 | **真名**/来历/能力/剧情下落 |
| 隐藏剧情·伏笔 | ~500 | 幕后真相、伏笔回收、跨作品互文 |
| 大事记时间线 | ~400 | 编年式速查 |
| 叙事基调·雷区 | ~250 | 画风、OOC 陷阱、时间线锚点 |

切入点条目：**只写该世界真实覆盖的阶位**，每阶七字段齐全，整条 ≥1500 字。**关键NPC 必须加粗真名**，禁止各阶复制同一句填充。

## 后续代码里程碑（Claude 负责，与内容生产并行不悖）

- **C1 卡片注入**：世界卡生成时，按 roll 中的世界名把 `·剧情`+`·阶位切入点` 精确注入 WorldSelector 请求
- **C2 正文注入**：玩家身处某世界时，正文 API 注入该世界 `·剧情`（不注切入点）
- **C3 公共区**：玩家 roll 后可上传条目到公共库（Cloudflare D1/DO，阶位→世界两级分类），他人生成世界时可选「自己 roll / 从公共库导入」
