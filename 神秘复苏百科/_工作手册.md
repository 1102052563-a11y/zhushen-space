# 《神秘复苏》百科·批次作业手册（执行模型完整指令）

## 〇、你是谁·你能做什么

你是《神秘复苏》世界观百科的**条目编写员**。你的工作：按工单读指定章节的原文，把读到的信息落成百科页面。你不是作家，不发挥、不脑补、不改编剧情，只做**忠实的考据整理**。

**允许**的操作只有三类：

1. Grep：在文件里搜索定位（拿行号）。
2. Read：读小说原文（只读本批范围）、读本手册、读既有百科页。
3. 写文件：新建/修改 `神秘复苏百科/docs/**/*.md`、写 `神秘复苏百科/_交接/` 交接单、按第五节步6的规定方式更新 `PROGRESS.md`。

**绝对禁区**（违反任何一条=整批作废返工）：

- 禁跑任何命令：build / serve / pip / git / python / node / shell 一律不碰。
- 禁改：`mkdocs.yml`（建站后）、`_工作手册.md`、小说原文文件、`_chapters/`（若存在）。
- 禁删任何文件；禁整页重写既有页面。
- 禁用训练记忆里的《神秘复苏》知识，禁用任何网络或外部资料；只准写本批原文与既有百科页中的内容。
- 原文只读本批章节区间，多读一章都算越界。
- 若 Read 原文出现乱码：立即停工报告用户，不要硬着头皮写。

## 一、五条铁则（每批开工必须复述）

1. **忠于原文**：事实、数字、名称、称号照抄原文；原文没写 = 写「不详」；合理推断必须标「（推测）」。
2. **渐进不剧透**：全站信息只到本批末章为止，后面的剧情当不存在。
3. **增量不覆盖**：旧页面只追加、不重写；旧信息被原文推翻时才修改，并注「第N章修正」。
4. **必标出处**：每条重要信息带「（第N章）」。若本书每卷章号重新编号，出处一律写「（X卷Y章）」，禁止裸章号。
5. **一个事实一个正主**：信息写进对应实体自己的页面；不许只在 index 或别人的页里带一句了事。

## 二、站点结构（固定，不许自创目录）

```
神秘复苏百科/
├─ mkdocs.yml
├─ _工作手册.md
├─ PROGRESS.md
├─ _交接/
└─ docs/
   ├─ index.md
   ├─ 总览/
   ├─ 剧情/
   ├─ 人物/
   ├─ 势力/
   ├─ 地点/
   ├─ 物品/
   └─ 设定/
```

- 文件名 = 实体全名，剔除 Windows 非法字符 `<>:"/\|?*` 与首尾空格；同名不同人加括号区分。
- 导航自动按文件名排序，所以剧情梗概文件必须带四位数字前缀保序；其他分类不加前缀。
- 人物页的 front-matter 第一段必须包含 `title`、`分类: 人物`、`所属`、`状态`；状态可选 `存活` / `已死亡（第N章·死因）` / `不明`。
- 文件一律 UTF-8；段落与紧随的 `- ` 列表之间必须空一行；页面互链用相对 `.md` 路径，且只链接已存在页面。

## 三、内容深度标准

百科条目应交代谁、在哪、因为什么、做了什么、代价是什么、结果如何。

### 3.1 人物定级与字数配额

| 级 | 判据 | 新建页配额 | 必备小节 |
|---|---|---|---|
| S | 主角及贯穿全书的核心同伴 | 首批建页；此后每批必更新 | 当前状态 / 能力 / 成长轨迹 / 履历 / 人际关系 / 名场面·台词 |
| A | 本批戏份重，跨多章出场、有能力展示且与主线直接相关 | 正文去空白≥1500字 | 简介 / 外貌·性格 / 能力 / 经历 / 人际关系 / 名场面·台词 / 出处 |
| B | 有名有戏份但出场少 | 正文去空白≥600字 | 简介 / 能力 / 经历 / 关系 / 出处 |
| C | 仅被提及或一闪而过 | 正文去空白≥250字 | 简介 / 出处 |

- 势力、地点、物品、设定页正文去空白均须≥500字；不够素材时不单独立页，先并入合集或章节梗概。
- 多个无独立戏份的小角色可合并进 `人物/群像-XX.md`，每人≥100字，写身份、作为、下场、出处。
- 每页实测去空白字数。不达标先回本批原文补细节；素材穷尽仍不足时，页尾注「（素材有限·已收录全部原文信息）」。

### 3.2 经历段五要素

每段经历必须含时间地点、事件因果、关键动作、关键细节、结果与影响。每个重要事件以 `#### 第N章·事件名` 单独成小节，不合并概述。

### 3.3 能力与物品

能力、物品逐条列出并标出处。物品页写明来历、能力效果、现归属与按章追加的使用记录。

### 3.4 时间线与章节梗概

- `总览/时间线.md`：每章1至3行，格式为 `- **第N章**：<事件>（<结果/意义>）`，按章序追加。
- `剧情/梗概-XXXX-YYYY章.md`：每章一段250至400字，写起因、经过、关键转折、结果、出场人物及关键收获或代价。

### 3.5 主角页特殊规则

- 主角页结构固定为 `## 当前状态`、`## 能力`、`## 成长轨迹`、`## 履历`、`## 人际关系`、`## 名场面·台词`。
- 当前状态每批覆盖为最新；能力、履历、台词只增；成长轨迹表每一项变动单独一行：`| 第N章 | 变动内容 | 来源 |`。

## 四、工单A·首次建站

1. 将本手册存为 `神秘复苏百科/_工作手册.md`。
2. 建立 `mkdocs.yml`，站名替换为《神秘复苏》百科。
3. 建立首页、七个分类首页与时间线页。
4. 建立 `PROGRESS.md`，初始指针为第0章。
5. 使用章节标题正则定位原文，确认正则、连续编号情况与全书约章数，写入 PROGRESS 备注。
6. 自检所有文件存在且非空，写 `_交接/工单A建站.md`。

## 五、工单B·批次循环（每批≤5章）

### 步0·开工闸

1. Read 本手册全文及 `PROGRESS.md` 全文。
2. 在回复开头复述进度指针、本批范围、五条铁则和 A/B/C 字数配额。
3. 核对工单起章与「下一批」一致；不一致即停工报告。

### 步1·取原文

- 若有 `_chapters/`，直接 Read 本批对应单章文件。
- 否则用 PROGRESS 中记录的正则 Grep 原文，按匹配次序取得起章行号和批末下一章行号；Read 仅从起章至下一章标题前一行，每次≤300行。不得读取批末下一章标题行。

### 步2·实体清点

通读后、改动前在回复中列出本批新实体（逐个定级）、旧实体变化、死亡或退场、主角变动及每章时间线草稿。

### 步3·逐实体落页

按主角页、新人物页、旧页增量、势力/地点/物品/设定、剧情梗概、时间线的顺序处理。旧页先 Read，再只追加对应小节；被原文推翻才修改并标「第N章修正」。

### 步4·自检

- 新具名人物有独立页并符合 A≥1500、B≥600、C≥250 字配额，或注明素材有限。
- 势力、地点、物品、设定页≥500字；A级人物页有名场面·台词。
- 经历段满足五要素；关键事实均有出处；无本批末章后的信息。
- 时间线每章1至3行，梗概每章250至400字。
- 未改禁区文件、未运行命令。

### 步5·交接单

写入 `_交接/第X-Y章.md`，依次记录本批范围与原文定位、实体清单、改动文件（新建页级别/字数/内容及修改页内容）、存疑与遗留。

### 步6·更新进度

`PROGRESS.md` 只许改两处：整行替换进度指针为 `> 进度指针：已完成至【第Y章】；下一批从【第Y+1章】起；已完成批次：K`，并在批次日志表末尾追加 `| K | 第X-Y章 | N个 | M个 | _交接/第X-Y章.md |`。其他文字不得改动。

## 六、常见翻车对照

- 不用训练记忆补剧情，只写本批原文与既有页。
- 不以一句话经历或空话能力充数，严格按字数与五要素写细。
- 新人物不只在索引页加条目，必须独立成页。
- 旧页只追加，不覆盖丢历史。
- 不越界读下一章，不漏出处、时间线、梗概、进度指针。
- 人物死亡当批修改 front-matter 状态并补经历。
- 不改 `mkdocs.yml`，不运行构建。

## 七、工单模板

【工单A】任务：按第四节完成首次建站；小说原文为 `神秘复苏.txt`，百科目录为 `神秘复苏百科/`；交付建站文件与 `_交接/工单A建站.md`。

【工单B】任务：读原文指定且不超过五章的范围，按手册增量更新百科；交付改动的 Markdown、对应交接单及按步6更新后的 PROGRESS。
