    总原则¶

    1. 忠于原著：只写正文明确写到的内容；推断须标注「（推测）」。
    2. 渐进不剧透：只记录已读章节的信息；未读到的不提前写。
    3. 增量更新：同一对象再次出场，在其已有页面追加，不新建重复页。
    4. 必标出处：关键信息注明相关章节，如 （第123章）。
    5. 命名：以原著正式名作页面名；别名写进 front-matter 别名。
    6. 关系（必备）：每个人物条目都应含「关系」一节，列出其与其他人物的关系，并用相对链接互链（NPC ↔ 主角、NPC ↔ NPC）；对方有专页则链页、无专页则纯文本注明。

    主角 / 常驻角色（会随剧情成长）¶

    苏晓这类持续成长、跨多个世界的角色，按以下规则维护，避免「覆盖即丢史」或页面变一坨：

    1. 当前值覆盖，历史只增不删：六维 / 等级 / 法力 / 乐园币 / 当前装备等「现状」覆盖到最新；每次变动同时往「成长轨迹」表追加一行（章节 ｜ 变动 ｜ 来源）。
    2. 区分两层：
      • 持久（跨世界带走）：基础六维、等级 / 阶位、天赋、法力、乐园币、规则物品、储物空间、已解锁功能。
      • 世界限定（留在该世界）：不可带出的装备、本地货币、临时身份 / 化名、任务世界 NPC 关系、世界之源。
    3. 两个聚合视角：
      • 角色页 = 纵向（时间）：简介 / 性格 → 当前状态（快照）→ 成长轨迹（表）→ 历程（按世界分块，每块精简 + 链到世界页）→ 持久关系。
      • 任务世界页 = 横向（空间）：该世界的人物 / 势力 / 地点 +「角色在本世界」（收获 / 世界之源 / 进度）+ 世界结束时的结算快照。
        • ⚠️ 结算快照铁律 ①·必须用「完整结构化」格式，不许只写几条 bullet 摘要。一份完整结算快照含三块：① 世界结算（难度 / 世界之源 % / 任务数 / 综合评价 / 奖励）；② 苏晓离世当下的「项 / 值」定格表——等级 / 阶位、职业、副职业、天赋、技能、法力、乐园币、属性点，外加独立的「六维」表（力 / 敏 / 体 / 智 / 魅 / 幸·标注有效值），再加 当前装备（离世时穿戴·武器 / 防具 / 饰品逐件列出·未穿戴的归入携出物）、携出物（规则物品 / 跨世界）、留在本世界（未带出） 三组清单；③ 随从 / 召唤物快照（见铁律③）。须注明「此为离世瞬间、与角色页『当前状态』不同」（如海贼王离世时尚无灭法之影职业；第六卷再入离世时六维仍是 32/32/23/32、乐园强化在此之后）。🔧 版式必须与既往世界逐字一致（模板＝ 火影忍者.md / 海贼王.md）：标题 ## 🏁 通关结算快照（X卷X章 · 离世瞬间，与当前状态不同）；### ① 世界结算＝| 项 | 值 | 表格（难度·获得世界之源·完成任务·综合评价·奖励·停留时长）；### ② 离世定格（X世界结束那一刻）＝| 项 | 值 | 表格（等级/阶位·六维·法力·核心技能·副职业·当前装备〔逐件〕·携出物·留在本世界·善后）；### ③ 随从快照＝无序列表。绝不要用 !!! success/quote/note 等 admonition 代替表格（已踩坑：全职猎人最初用 admonition 写、被要求返工改成表格）。
          • 🔒 强制规则·每个任务世界通关 / 离开时都必须补这份完整结算快照，无一例外——含晋升任务世界（钢之炼金术师·评价 B）、独立试炼 / 生存场景（如 T12K 星球·吞噬岛）等「无主线」的特殊世界也照补（这类世界只是「完成任务数 / 评价」一栏特殊，其余照写）。已通关世界若缺快照即视为待补（海贼王首入 / 东京喰种 / 进击 / 斩赤 / 寄生兽 / 海贼王再入 / 钢之炼金术师均已补齐）；今后每读到一个世界通关、当批就把该世界的完整结算快照补上，不得遗漏、不得只写摘要。离世定格用「世界结束那一刻」的数值（返乐园后的属性强化 / 新装备 / 技能升级一律不算进定格、只在备注里说明发生在此之后）。
        • ⚠️ 结算快照铁律 ②·同一任务世界被多次进入 → 每次各记一份独立结算快照、用副标题（## / ###）明确区分（如「结算快照 · 海贼王首入（第22章）」与「结算快照 · 海贼王第六卷再入（六卷70章）」），绝不混为一谈、也不互相覆盖。
          • 进一步·拆成独立页 + 按剧情顺序排 nav：若同一世界两次进入的内容都很多，应各自独立成页（如 海贼王.md〔首入〕与 海贼王-第六卷再入.md），并在 mkdocs.yml 的「任务世界」导航里按剧情发生顺序分别列出（首入排第 1、第六卷再入排在寄生兽之后第 6），绝不因是同一世界就并排靠在一起。同理 任务世界/index.md 也按 ①②③… 剧情序各列一条。
        • ⚠️ 结算快照铁律 ③·随从 / 召唤物：一个角色一页、绝不合并。苏晓名下每个随从 / 契约生物 / 召唤物各自单开一页（与「人物建页铁律」一致·放 docs/人物/·如头号随从 布布汪），在 mkdocs.yml「主角与随从」分组下逐条列出（新随从在布布汪之下添加新条目）；页内记来源 / 能力 / 归属 / 历程，并在每个世界结束时补一份与主角同格式的结构化结算快照（项 / 值表：形态/品级·属性·技能·本世界作用·去向〔带出 / 留下 / 阵亡 / 解约〕）。随从快照详情归该专页、世界页只摘要 + 链接（与苏晓本体分开记，避免淹没主页）。
    4. 关系只有一个正主：世界 NPC 的详情写在该 NPC 页，角色页 / 世界页只链接、不复制。
    5. ⚠️ 人物建页铁律·每个具名人物都单开一页：凡是有名字的人物 / 人造人 / 剧情角色 / 契约者 / 盟友 / 反派（哪怕只出场一两次），都必须各自单独建一个 .md 页（放 docs/人物/），并在 mkdocs.yml 的「人物」导航里按其所属世界分组列出（海贼王 / 东京喰种 / 进击的巨人 / 斩·赤红之瞳 / 寄生兽 / 钢之炼金术师 / 轮回乐园·现实 …）。
      • 人物/index.md 只作「按世界的索引 + 一句话简介 + 链接」，不再把人物详情留作裸 bullet——详情一律落到各自的人物页。
      • 新出场人物立即建页：每批阅读遇到新具名人物，当批就为其建页 + 进 nav（绝不只在 index.md 写一条 bullet 了事）。
      • 页内格式：front-matter（title / 分类:人物 / 所属世界 / 身份 / 状态〔在世·殁·提及〕）+ 简介 / 身份 + 能力 / 经历 + 与苏晓的关系 + 出处章节；殁者标「（殁·N卷N章）」。
      • 背景概念（如「恶魔族」「不死军团」「合成兽」这类群体 / 设定而非具体人物）可不单独建人物页、归入 生物·万族 或对应世界页。
      • 存量回填：早期只写在 人物/index.md 的 bullet 人物（海贼王第六卷再入 / 寄生兽 / 钢之炼金术师 / 现实复仇线等）须逐世界补建独立页（可分多批进行·从当前所在世界优先）。
