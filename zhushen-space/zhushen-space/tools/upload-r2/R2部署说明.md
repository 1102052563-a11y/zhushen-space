# R2-only 图片部署

把强化看板娘(`图片/`)、欢愉宫(`欢愉宫图片/`)的立绘改为 **存 Cloudflare R2、git 不入库**，由 Pages Function 在原路径同源提供，**前端代码无需改动**。

```
本地 图片/<角色>/阶段N/*.png
   ──npm run upload-r2──▶ R2 桶: enhance-bosses/<角色>/阶段N/*.png + manifest.json
                                          ▲
zhushen-space.pages.dev/enhance-bosses/*  ─┘  (functions/enhance-bosses 从 R2_BUCKET 取)
```

---

## 一、你在 Cloudflare 做的（一次性）

1. **建 R2 桶**：R2 → 创建桶，名字如 `zhushen-assets`（要绑卡才能开通 R2，但全程在免费额度内 = 0 花费）。
2. **把桶绑到 Pages 项目**：`Workers & Pages` → 你的项目 → **Settings → Bindings → Add → R2 bucket**（2026 新版位置；旧版在 Functions 下）：
   - 变量名（Variable name）：**`R2_BUCKET`**（必须一字不差，Function 里用的就是它）
   - R2 bucket：选 `zhushen-assets`
   - 加完**重新部署一次**才生效
3. **建 R2 API 令牌**（给上传工具用）：R2 → Manage R2 API Tokens → Create，权限选 **Object Read & Write**，记下：
   - **Access Key ID** / **Secret Access Key**
   - **账号 ID**（R2 概览页，或 S3 端点 `https://<账号ID>.r2.cloudflarestorage.com`）
4. 确认 Pages 的「构建根目录(Root directory)」是 `zhushen-space/zhushen-space`（`functions/` 在这里，才会被部署）。

## 二、本地准备（一次性）

```powershell
cd C:\Users\Administrator\Desktop\前端卡\files\zhushen-space\zhushen-space
npm i -D @aws-sdk/client-s3
```
把 `tools/upload-r2/config.json` 填好（已 gitignore，不会入库）：
```json
{ "accountId": "...", "accessKeyId": "...", "secretAccessKey": "...", "bucket": "zhushen-assets" }
```

## 三、上传

```powershell
npm run upload-r2
```
会把 `图片/`、`欢愉宫图片/`、以及**背景音乐**推到 R2，并生成各自的 `manifest.json`（`enhance-bosses/`、`joy-girls/`、`audio/bgm/`）。默认跳过已存在，只传新增。

### 背景音乐（BGM）说明
- 源：仓库根 `BGM/`（已 gitignore，1.3GB 不入库）。**当前只上《2077》一套**（`upload-r2.mjs` 里 `BGM_SRC` 指向 `BGM/游戏bgm/2077`）；要全上或换目录，改 `BGM_SRC = join(HERE, '../../../../BGM')`（会递归所有子文件夹）。
- R2 键：`audio/bgm/<相对路径>` + `audio/bgm/manifest.json`（数组格式 `[{file,name,category,bytes}]`）。
- 线上由 `functions/bgm/[[path]].js` 在同源 **`/bgm/*`** 提供（Function 内把 `/bgm/*` 映射到 R2 键 `audio/bgm/*`；**支持 Range**）。前端 `bgmBase()` 默认走 `/bgm`，无需 CORS。
  - ⚠**不能用 `/audio/bgm/`**：`/audio/` 下有音效 wav 静态文件，Pages 会把整个 `/audio/*` 当静态资源、绕过 Function（曾导致 BGM 全 404 掉 SPA、请求飙升）。
- ⚠️ 版权：游戏 OST 属商业版权，公开部署风险自负。

## 四、部署 & 验证

1. commit + push（`functions/` 会随 Pages 一起部署）。
2. 浏览器访问验证：`https://zhushen-space.pages.dev/enhance-bosses/manifest.json` 应返回 JSON；随便点一张图 URL 应能显示。
   - BGM 同样验证：`https://zhushen-space.pages.dev/bgm/manifest.json` 应返回曲目数组（**不是 HTML**）；随便点一个音频 URL 应能播放。
3. 进游戏看强化看板娘/欢愉宫立绘是否正常加载；封面/游戏内左下角出现「🎵 开启背景音乐」，点开始播。

## 五、最后一步：让 git 真正不存图（**确认 R2 工作正常后再做**）

```powershell
git rm -r --cached 图片 欢愉宫图片   # 从 git 移除（本地文件保留）
```
然后在 `.gitignore` 末尾加：
```
/图片/
/欢愉宫图片/
```
commit。此后仓库不再随图膨胀，构建也变快。
> ⚠️ 先确认 R2 那边图都在、能访问，再做这步——否则线上会丢图。

---

## 日常工作流（以后）

1. 用 NAI 工具生成图到 `图片/<角色>/阶段N/`；
2. `npm run upload-r2`（增量重传，会覆盖同名）；
3. push（若改了 Function/前端）或直接生效（纯加图时 manifest 已在 R2，刷新即见）。

## 注意

- Pages Function 免费额度 10 万请求/天，个人项目够用。
- `R2_BUCKET` 这个绑定名前端 Function 写死了，改名要同步改 `functions/*/[[path]].js`。
- dev 本地（`npm run dev`）仍走 `public/enhance-bosses/`（vite 插件从本地 `图片/` 同步）——所以本地照常能看，线上走 R2，两不影响。
