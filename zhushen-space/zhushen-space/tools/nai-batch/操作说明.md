# NAI 批量生图 · 操作说明

凯莉等角色分阶段立绘的批量/随机生图工具。两个入口：**网页工作台**（你自己调）+ **命令行 CLI**（我帮你跑）。
位置：`zhushen-space/zhushen-space/tools/nai-batch/`

---

## 0. 分工约定（重要）

- **正向**：我负责（身份块、池子里的标签）。
- **负向**：你负责。**我不写、不改、不看你的负向**。
- 我帮你跑时，用命令行参数覆盖「文件夹/前缀/张数/起始号」，**不读你 `jobs.json` 里的池子和负向内容**，CLI 内部直接用你保存的那套配置。

---

## 1. 启动网页工作台

在内层目录 `zhushen-space/zhushen-space/` 下：

```powershell
npm run nai-ui
```

看到 `NAI 生图工作台 → http://localhost:5188` 即成，浏览器打开该地址。
> **想让它不掉线**：自己开一个终端跑上面这条命令，**别关那个窗口**（这样服务独立于 AI 会话，不会被回收）。

---

## 2. 网页工作台用法

| 区块 | 说明 |
|---|---|
| ① 输出位置 | 文件夹（如 `凯莉/阶段4`）、前缀（如 `kaili_s4`）、输出根、尺寸、起始序号 |
| ② 正向身份块 | 所有图共用，保人物一致；折叠区可改「画师/质量串」（自动前置） |
| ③ 负向 | **你自己填**（我不碰） |
| ④ 双随机正向池 | **池1**＋**池2**，每张各随机抽 `drawCount` / `drawCount2` 条接到正向后；**机械随机不判断兼容** |
| 定量/高级 | 数量（非连续时）、种子、间隔、姿势列表、模型/步数/scale/采样器 |

**按钮**：
- **▶ 生成**：按「数量」出图；勾「连续模式」则不停生成到点停。每张结果实时预览、自动存盘，图下方青字显示这张抽中的标签。
- **💾 保存到服务器**：把网页当前所有字段（含两个池、负向、抽取数…）写回 `jobs.json`。**改完池子务必点一下**，否则命令行（我帮你跑）读到的还是旧的。
- **↺ 恢复默认**：清掉本地修改，从 `jobs.json` 重新载入。

> 你在网页里改的内容**自动存浏览器、刷新不丢**；但要让命令行也用上，必须点「保存到服务器」。

## 3. 命令行 CLI

跑 `jobs.json` 里的任务（我通常用这个帮你跑）：

```powershell
node tools/nai-batch/nai-batch.mjs [参数]
# 或 npm run nai
```

**参数**（可叠加；带 `--xxx=值`）：

| 参数 | 作用 |
|---|---|
| `--count=N` | 覆盖张数 |
| `--start=N` | 覆盖起始序号（续号、不覆盖已有图；脚本本就跳过已存在文件） |
| `--folder=凯莉/阶段4` | 覆盖输出文件夹 |
| `--prefix=kaili_s4` | 覆盖文件名前缀 |
| `--dry` | 演练：只打印计划+抽样，不调接口 |
| `--list` | 列出任务清单 |
| `--max=N` | 每个任务最多生成 N 张（冒烟测） |
| `--force` | 覆盖已存在的图 |
| `--job=子串` | 只跑 folder/prefix 含该子串的任务 |

> 用 `--folder/--prefix/--count/--start` 这套，就能**不读你的 jobs.json 内容**、只用你保存的池子和负向跑。

## 4. 常用操作示例

**给某阶段续生成 N 张**（自动从该文件夹当前最大号接着排）：

```powershell
$dir = "C:\Users\Administrator\Desktop\前端卡\files\图片\凯莉\阶段4"
$max = 0
if (Test-Path $dir) { $nums = Get-ChildItem $dir -Filter "kaili_s4_*.png" | ForEach-Object { [int]($_.BaseName -replace 'kaili_s4_','') }; if ($nums) { $max = ($nums | Measure-Object -Maximum).Maximum } }
node "C:\Users\Administrator\Desktop\前端卡\files\zhushen-space\zhushen-space\tools\nai-batch\nai-batch.mjs" --folder=凯莉/阶段4 --prefix=kaili_s4 --count=20 --start=($max+1)
```

**先演练看抽样**：在上面命令尾部加 `--dry`。

## 5. 输出与部署

- 图落地 `图片/<角色>/阶段N/`（N=1~4），文件名 `<prefix>_NN.png`。
- **vite 插件**在 `build` / `dev` 启动时，自动把 `图片/<角色>/阶段N/` 同步到 `public/enhance-bosses/<角色>/阶段N/` 并重写 `manifest.json` → 角色成为**强化看板娘**（分阶段立绘）。
- 想生效进游戏：重启 dev 或 `vite build`。
- ⚠️ 图是入 git 的：几百张没问题；上千张考虑 **Cloudflare R2**（图存 R2、前端存 URL，git 不膨胀）。

## 6. 故障排查

| 现象 | 原因 / 解决 |
|---|---|
| 网站打不开 | 服务进程被回收。重启 `npm run nai-ui`；**自己开终端跑最稳**。 |
| 改了池子刷新就还原 | 已修（localStorage 自动保存）。想同步到命令行要点「💾 保存到服务器」。 |
| 池2 没生效 | ① 网页池2 框可能空了 → 点「恢复默认」重载；② 网页改了没点「保存到服务器」→ 命令行用的还是旧池子；③ 调大「池2 抽取数」。 |
| 网页生成的图跟提示词对不上 | 已修（曾因 `positive`/`prompt` 字段名不匹配导致正向被丢，现已对齐）。 |
| 中文路径在 PowerShell 里显示乱码 | 只是控制台编码，**文件内容是 UTF-8、正常**。 |
| 两个进程同时打 NAI 报 `429 Concurrent generation is locked` | 别并发跑；一批跑完再开下一批。 |

## 7. 关键文件

| 文件 | 作用 |
|---|---|
| `nai-batch.mjs` | 命令行批量生成（含双池随机、ZIP 解码、超时重试） |
| `server.mjs` + `ui.html` | 网页工作台（GUI + 后端 API） |
| `jobs.json` | 任务/池子/负向配置（网页「保存到服务器」写它；CLI 读它） |
| `config.json` | NAI token + 默认参数（**已 gitignore，勿外传**） |
