## 🎨 新功能：HP/EP 条「自定义称呼」（换皮）

> 日期：2026-06-17 ｜ 状态：**已实现**（vite build 通过 + preview 实测换皮生效）。

**需求**：玩某些特殊角色时，HP 条 / EP 条显示该角色专属称呼（如吸血鬼 HP=「血池」、EP=「血怒」；名字**任意，可能很奇怪**）。**纯展示换皮，不改任何数值逻辑**——HP=体质×20 / EP=智力×15 上限、战斗、骰子、治疗、`<state>` 指令全部不动。

**最高铁则（违反必出 bug）**：自定义名**「只出不进」**——只用于 ① 渲染到条上、② 作为一句别名提示喂给 AI 让正文措辞跟着叫；**它永远不进入任何解析器 / 正则**。机器通道（`<state>` 指令 `hp.B1`/`mp.B1` + 正文「当前HP：X/Y」状态行）**永远写死 HP/EP 原词**，与自定义名无关。
> ⚠️ 已否决方案：让兜底扫描器 `applyNarrativeVitals` 用自定义名拼正则去抓「当前血池：X/Y」。**不要这么做**——名字任意 + AI 措辞会飘，让前端"往回解析自定义名"天生脆。正确做法是用提示词约束 AI：状态行 / 指令一律 HP/EP，自定义名只出现在散文里。

**实际改动（5 处）**：
- 数据：`store/playerStore.ts` `PlayerProfile` 加 `hpLabel?` / `epLabel?`（optional，随 persist + saveManager 自动进存档；留空=默认）。
- 输入：`PlayerSidebar.tsx` **血条下方小按钮「✎ 自定义血条」**（`labelOpen` 开关）点开，折叠面板含 HP条 / EP条 名称输入（`onChange`→`setProfile` 实时改名）+ 血条皮肤选择（见下一节）。*默认收起、不占资料列表。*
- 渲染：`PlayerSidebar.tsx` 底部双条 `label={profile.hpLabel || '生命 HP'}` / `|| '蓝量 EP'`。
- 注入：`systems/structuredRecall.ts` `serializePlayerCard` 的 HP/EP 行——**设了别名才**追加`；正文叙述称「X」，状态行/指令仍写 HP/EP`（`HP:` 数值锚点不变）。
- 提示词：`promptRules.ts` `HPEP_NARRATIVE_ONLY_RULE` 末尾加一条「散文用别名，状态行与 `hp.<id>`/`mp.<id>` 指令一律 HP/EP」。

**未做（有意）**：`StatusBar.tsx` 是死代码（全仓库无 `<StatusBar>` 引用）→ 跳过；NPC（`NpcDetail`）换皮后续再说（NPC 多，更适合 AI 配名而非手填）。`applyNarrativeVitals` / `hp.<id>` 指令解析 / 战斗结算 **一行未碰**。

**preview 实测**：在主角资料里把 HP条称呼 设为「血池」→ 侧栏 HP 条标签即从「生命 HP」变「血池」，EP 条不受影响仍「蓝量 EP」；清空即回退默认。（测试值已还原。）

## 🎨 新功能：血条 / 蓝条皮肤（20 款换肤）

> 日期：2026-06-17 ｜ 状态：**已实现**（vite build 通过；CSS 样式画廊已确认 10 款视觉与动画）。

主角侧栏「✎ 自定义血条」面板里加**血条皮肤**选择网格：10 款，2 列，每格是该皮肤的**实时迷你 HP/EP 预览**，点格即换、侧栏血条实时更新。**纯展示外观，不碰 hp/ep 数值与指令通道。**

10 款：经典(立体管状) / 霓虹(流光辉光) / 熔岩(余烬漂移) / 冰晶(棱面流光) / 毒液(气泡上浮) / 黄金(厚金流光) / 赛博(分段扫描) / 血雾(红蓝脉冲) / 星河(流动闪烁) / 等离子(色相流转)。**每条另叠通用三层：玻璃高光 + 前沿能量亮边 + 斜向流光扫过。**

第二批 5 款（网络灵感原创·CSS 自写不抄素材）：条纹(理发店斜纹流动) / 波涛(液体水面波纹横流) / 格栅(能量分块呼吸) / 聚能(亮端渐强充能脉冲) / 极光(柔彩流动)。新增 keyframes `barStripe/barWaveMove`；皮肤选择网格加 `max-h-[46vh] overflow-y-auto`（皮肤多了不撑爆面板）。

第三批 5 款（网游血条风·原创复刻风格特征，**不嵌任何游戏图片素材**）：翠盟(MMO 翠绿生命/蔚蓝法力,WoW·LoL 配色) / 魔渊(暗黑破坏神·液体宝珠玻璃穹顶高光) / 街机(格斗黄橙红血槽 + 青色超必杀槽脉冲) / 魂界(魂系暗血红极简 + 顶端金线饰边) / 刻度(MOBA·Boss 黑色分段刻度线)。复用 `barTwinkle`、无新 keyframes。**现共 20 款。**

两项交互增强：① **点血条本身**即展开「✎ 自定义血条」面板（`PlayerSidebar` 把双条裹进 `onClick=setLabelOpen(true)` 容器，原小字按钮保留）。② **血量过低告警**——`Bar.tsx` 算 HP ≤ 20% 给 fill 加 `.bf-danger`（`index.css`），**覆盖任何皮肤**转红急促闪烁(`@keyframes barDanger`)，回血到 20% 以上自动恢复原皮肤；`prefers-reduced-motion` 下只静态红、不闪。仅作用 HP 条。

**实际改动（4 处）**：
- 样式：`src/index.css` 的 `.barfill` + `bf-<style>-hp/-ep`（10×2）。**多层结构**：fill 背景(渐变/纹理) → `::before`(玻璃高光+前沿亮边,全款共享) → `::after`(斜向流光扫过,全款共享)。keyframes：`barSweep/barFlow/barFlicker/barScan/barHue/barTwinkle/barEmber/barBubble/barPulseRed/barPulseBlue` + `@media(prefers-reduced-motion)` 关闭动画。**结构关键**：外层 track 去 `overflow-hidden` 让辉光外溢；fill 自身 `overflow:hidden` 裁切高光/流光/纹理层。
- 组件：`components/Bar.tsx` 加 `styleId` / `kind`('hp'|'ep') props + 导出 `BAR_STYLES`；给了 `kind` 走新结构，否则维持旧行为。
- 数据：`store/playerStore.ts` `PlayerProfile` 加 `barStyle?`（随存档；空=经典）。
- 接线：`PlayerSidebar.tsx` 双条传 `styleId={profile.barStyle} kind=...`；面板加皮肤选择网格。

**注意**：`components/Bar.tsx` 仅 PlayerSidebar(HP/EP) 与 StatusBar(死代码) 使用；AdventureTeam/Combat/SubProfession 用的是各自文件里的同名局部 Bar（prop 是 cls/pct/track，不同签名），未受影响。

## 🆕 新功能：肖像图库

开发者把肖像图按分类丢进 `public/portraits/<分类>/`，玩家在游戏中给主角 / NPC 选肖像。

- **文件夹即分类**：子文件夹名 = 分类，文件名（去扩展名）= 显示名；直接放根目录 = 无分类。
- **manifest 自动生成**：`vite.config.ts` 新增插件，在 build（含 Cloudflare）和 dev 启动时自动扫描生成 `public/portraits/manifest.json`（已 gitignore，勿手改）。
- **存 URL 不占存档**：选中后存的是图片网址，不占体积、多角色可共用。
- **入口**：主角立绘 / NPC 头像旁的「📁 图库」按钮；**点空白肖像也直接打开图库**；弹窗顶部按分类出筛选标签。

涉及文件：`PortraitPicker.tsx`(新) · `public/portraits/`(新) · `vite.config.ts` · `.gitignore` · `PlayerSidebar.tsx` · `NpcDetail.tsx` · `OnScenePanel.tsx`
