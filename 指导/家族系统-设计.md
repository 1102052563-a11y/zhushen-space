# 家族系统 · 设计（待实现）

> 记录于设计讨论阶段。核心结论：家族 = **异步·跨存档的玩家契约者战队**，重度复用 `仓库 / ChatDO / 世界竞技场 / 丰碑`，新增 `GuildDO`。
> ⚠**家族据点建筑（用户铁律·同商店货品套路）**：**成员自定义**（自由写名/描述/效果/立绘）＋**接 API 按提示词生成**（输入提示词→生成→可编辑）＋**成员集资建设升级**；但**绝不从游戏正文自动生成**（AI 生成是显式提示词输入·非读正文）。
> **最重要的一条判断**：本游戏是「单机 AI RPG 内核 + 异步社交外壳」，所以**不做 WoW 式要全员同时在线的实时公会副本，做异步家族**——Clash 部落/Torn 帮派/FFXIV 部队证明异步公会本就是留存最狠的形态（捐赠软义务、连击冲榜、金库/增益全是异步）。

---

## 一、核心定位：家族 = 唯一的「玩家↔玩家」组织

与现有三层严格区分（别混，别污染各自体系）：

| | 🏰 家族（本设计·**新**） | 🛡 永久冒险团（`adventureTeamStore`） | 🤝 临时世界队伍 | 🏛 势力（`factionStore`） |
|---|---|---|---|---|
| 成员 | **真实玩家**（Discord uid） | 自己的团员 NPC | 临时队友 NPC | AI 世界里的 NPC 势力 |
| 范围 | **跨存档·账号级·常驻** | 跨世界永久（本档） | 仅当前世界 | 剧情演化 |
| 目的 | 归属 / 共享进度 / 社交义务 / 家族战 | 个人带队晋级 | 一起刷个本 | 世界观 |
| 后端 | **GuildDO（新）** | 本地 store | NPC 字段 | 本地 store |

一句话：**冒险团是"你带的 NPC 队"，家族是"你加入的真人组织"**，两者并存（你有自己的冒险团，同时属于某个玩家家族）。家族身份绑 Discord uid，同聊天室/商城。

## 二、架构落地（全落在现成基建上）

```
GuildDO（每家族一实例 · idFromName(guildId) · 仿 RoomDO）
  = 名册/军衔 · 共享贡献&等级 · 金库 · 周任务 · 据点快照 · 家族频道 · 编年史
GuildListDO（单例 · 仿 LobbyDO/AssistDO）
  = 家族注册表：创建/搜索/申请加入 · 公开家族卡（名/徽记/等级/人数/宣言/招募中）
身份 = verifyChatToken → pid=chat:<uid>（同 assist/shop/chatroom）
```

**复用清单（不重造轮子）：**

| 家族要的 | 直接复用 | 备注 |
|---|---|---|
| 家族据点（共享基地） | `territory`（领地） | 领地升级/建筑体系照搬；权属从"个人"扩到"家族共享"，会长/长老可建 |
| 家族金库（共享仓库） | `vault`（账户仓库） | ⚠必带 [[account-vault-and-stash-payload]] 的"完整快照"铁则防丢词缀；权限按军衔+每日额度 |
| 家族频道 | `ChatDO` 范式 | 一个 scope=guildId 的聊天流（或 GuildDO 内自带 WS 广播） |
| 家族战（团体竞技） | `world-arena`（ArenaWorldDO 服务端裁判） | 各家族出成员卡，服务端"战力+种子纯函数"聚合，异步出结果 |
| 家族编年史/丰碑 | `monument`（纪念丰碑·R2 云同步） | 家族大事记 + 阵亡/退隐英灵，跨存档 |
| WS 客户端/协议/live store | `shopClient/shopProtocol/shopMarketStore` 三件套范式 | 照搬 connect/重连/心跳/dispatch |
| 上传/消毒/防刷 | `AssistDO/ShopDO` 的 sanitize + MIN_INTERVAL + MAX 上限 | Hibernation ≈$0 |

## 三、数据结构

### 前端 `guildStore`（drpg-guild · 账号级 · 部分持久化）
```ts
interface GuildSummary {           // ← 持久化：让单机侧离线也能应用家族 buff + 注入正文
  id: string; name: string; tag: string; emblem?: string;   // 徽记 = emoji 或缩略图
  role: GuildRank;                 // 我的军衔
  level: number;
  perks: GuildPerk[];              // 已解锁增益（buff 注入用）
  joinedAt: number;
}
type GuildRank = 'leader' | 'viceLeader' | 'elder' | 'member';
interface GuildPerk { key: string; label: string; value: number; }   // 如 {key:'expBoost',value:0.05}

interface GuildState {
  my: GuildSummary | null;         // 持久化（partialize 只留这个）
  // ── 以下 live 态·不持久化·连 GuildDO 时填 ──
  roster: GuildMember[]; chest: any[]; weeklyTasks: WeeklyTask[];
  contrib: Record<string, number>; chronicle: ChronicleEntry[];
  base: any;                       // 家族据点=领地快照
  status: 'idle'|'connecting'|'connected'|'closed'|'error'; online: number; error: string | null;
  _set / reset / ...;
}
```
- **持久化只留 `my`**（家族摘要 + 已解锁 perks），供离线注入 buff/正文；live 细节（名册/金库/任务）连 GuildDO 时拉。
- 纳入 `saveManager`？——**不纳入**（账号级·跨存档·不随存档快照，同 joyStore/monument 口径），新游戏保留家族身份。

### 后端 GuildDO storage
```
guild = {
  id, name, tag, emblem, manifesto, recruiting,
  ownerId, createdAt,
  level, exp,                         // 贡献累计 → exp → level
  perks: [{key,label,value}],         // 按 level 解锁
  members: [{ pid, name, rank, contribTotal, contribWeek, joinedAt, lastActive, du, avv, ds, nc }],
  applicants: [{ pid, name, at }],    // 申请加入队列（会长/长老审批）
  chest: [ ...item snapshots... ],    // 家族金库（带完整快照）
  weekTasks: { weekId, goals:[{key,target,cur,reward}], claimed:[pid] },
  contribLog / chronicle: [ ... ],    // 编年史
  baseSnapshot: { ...territory... },  // 家族据点
}
```

### 线协议 `guildProtocol.ts`（仿 shopProtocol）
- **Inbound**：`hello / guild_synced(全量) / member_joined / member_left / rank_changed / contrib_bumped / task_progress / chest_changed / chronicle_added / level_up / rate_limited / error`
- **Outbound**：`create_guild / apply / approve / kick / set_rank / contribute(kind,amount,detail) / deposit / withdraw / edit(name/tag/emblem/manifesto/recruiting) / disband`
- **GuildListDO**：`list_guilds / search / apply`（公开家族卡）

## 四、核心循环：贡献 → 等级 → 增益（FFXIV FC + Clash 周任务）

1. **贡献值来源**（玩家正常玩就产生·确定性上报，不经 AI）：
   - 击杀强敌 / 通关世界（按世界之源%或评级折算）/ 捐乐园币·魂币 / 捐物进金库 / 完成家族周任务。
   - 上报口径：前端在既有结算点（`world-settlement` / `gemDrop` / 货币 `adjustCurrency`）附带 `guildClient.contribute(kind, amount)`；GuildDO 累加到该成员 `contribTotal/contribWeek` + 家族 `exp`。
2. **家族等级** = exp 阈值表（如 Lv1→Lv2 需 5000，指数增长）；升级广播 `level_up` + 进编年史。
3. **家族增益（GuildPerk·按等级解锁）**——**注入核心循环**才有留存力：
   | 等级 | 增益示例 |
   |---|---|
   | Lv2 | 全员每日家族津贴（乐园币）|
   | Lv3 | 世界结算评级小幅加成 |
   | Lv4 | 金库容量↑ + 传送/入世费减免 |
   | Lv5 | 掉率/合成品质小幅加成 |
   | Lv7 | 家族专属称号 + 正文 buff（士气）|
   - ⚠**P1 只实现 1~2 个安全 perk**（家族津贴 = 每日发币 + 结算评级+X%·各一个 hook），其余先只解锁展示、逐期接。**忠于"别凭空加数值"铁律**：perk 幅度小、确定性、可关。
4. **周贡献任务（Clash 软义务·留存核弹）**：
   - 家族级周目标（累计击杀 X / 通关 Y 世界 / 捐 Z 币），`weekId` 每周重置 = **每周回来的理由**。
   - 完成 → 家族 exp + 全员可领奖（`claimed` 去重）。
   - **贡献榜**（周/总）+ 活跃度（lastActive）；**软义务**靠会长手动踢人（Clash 口径：游戏不硬罚，家族自定门槛自治）。

## 五、军衔与权限（确定性·会长权威）

| 军衔 | 权限 |
|---|---|
| 会长 leader | 改家族设置/徽记/宣言 · 审批/踢人 · 任命军衔 · 建据点 · 解散 · 金库无限取 |
| 副会长 viceLeader | 审批/踢成员 · 建据点 · 金库高额度 |
| 长老 elder | 审批加入 · 金库中额度 |
| 成员 member | 贡献 · 聊天 · 金库低额度存取（按容量+每日额度） |

- 一人一家族（换家族先退会）；家族人数上限（如 30，随等级↑）。
- 会长退会/长期不活跃 → 自动传位（副会长>贡献最高）。

## 六、竞争层（Clash 战 + Torn 连击 · 均异步）

- **家族战**（复用 `world-arena`）：定期（周）配对两家族，各出 N 张成员卡，服务端"战力+种子纯函数"自动对战聚合总分，高者胜 → 家族 exp/称号/榜。**全异步**，无需同时在线。
- **家族连击**（Torn chain）：开一个活动窗口，全员 击杀/通关 累计冲**里程碑**（10/50/200/1000…），每档发家族奖励。协同但异步（各自玩、累计）。
- **家族排行榜**（GuildListDO 排序）：按 等级/周贡献/战绩；上榜=身份=冲榜动力。

## 十、关键设计决策 & 坑

1. **异步优先**：所有机制都能"各自玩、累计/聚合"，绝不要求同时在线（本项目多人是房主权威回合制，重；家族走看板/累计式）。
2. **软义务而非硬罚**（Clash 口径）：游戏不因不贡献而惩罚，家族自定门槛 + 会长踢人自治——最粘且最省实现。
3. **忠于数值铁律**：家族 perk 幅度小、确定性、可关，别让 buff 变成凭空数值膨胀（对齐 [[faithful-to-narrative-no-stat-inflation]]）。
4. **账号级·不进 saveManager**：家族身份跨存档（同 joyStore/monument）；`guildStore.my` 持久化供离线 buff/正文，live 态连 DO 拉。
5. **成本≈$0**：GuildDO/GuildListDO 全走 WebSocket Hibernation + sanitize + MIN_INTERVAL 防刷 + MAX 上限。
6. **必须 deploy**：新增 `GuildDO`/`GuildListDO` → wrangler 加绑定 + migration v7；`multiplayer-worker` 目录 `npm run deploy`（同 [[cloudflare-deployment]]）。
7. **别和 势力/冒险团 混**：家族是玩家↔玩家；势力是 AI 世界 NPC 组织；冒险团是个人 NPC 队。三者独立 store、独立面板。
