# 战斗系统重置 · 设计与实现计划

> 锁定设计文档。功能定位见 `docs/FEATURES.md`，代码定位见 `docs/CODE_MAP.md`。
> 参考来源：SillyTavern 卡 **TGv3.2**（武侠·前端算+AI叙事）与 **6.png 终末之诗**（Kenshi·骰子锚定+描写规则）。

## 0. 锁定决策（用户拍板）

| # | 决策 | 选定 |
|---|---|---|
| 1 | 能量模型 | **沿用 EP 蓝条**做技能消耗，战斗内每回合小幅自动回 |
| 2 | 伤害来源 | **混合**：AI 只填倍率/层数/回合，前端 ×攻击力档 放大 |
| 3 | 命中 | **STS 式必中**（默认不 miss，更直观）|
| 4 | 替换范围 | **(A) 整套替换** combatEngine 内核 |

## 1. 核心定位

> 战斗 = 前端一个确定性「标签虚拟机（Tag VM）」跑完整场，敌人用本地启发式 AI（**零 API**），结束后把战斗日志压成一行战报，**只调一次 AI** 润色成正文。

与旧系统最大区别：旧系统每个 NPC 回合调一次 LLM 决策 + 每个动作再调一次 LLM 叙事（等待久）。重置后**战斗中 0 次 API，仅结束润色 1 次**。

## 2. 三段式管线

```
玩家正文/行动 → 前端战斗引擎（动作→标签结算→buff/debuff→HP/护盾，本地敌人AI 驱动回合循环，0 API）
            → 战斗日志 → BATTLE_RECORD 压缩战报 → AI 一次润色（按人设·禁改数值）→ 正文衔接
```

## 3. 战斗原语（标签系统 = 心脏）

技能不再是自由文本，而是「标签 + 参数」的集合。前端只认枚举里的标签，算不出的直接忽略 → 技能永远跑不出战斗系统。

### 3.1 动作层（最外层固定 5 个）
| 动作 | 消耗 | 行为 |
|---|---|---|
| 普攻 | 0 | 内置 `[{tag:deal, mult:1.0}]`，走武器/基础攻击 |
| 技能 | EP | 执行该技能 `effects[]` 标签列表 |
| 道具 | 1 个 | 执行道具 `effects[]`（同一套标签）|
| 防御 | 0 | 内置 `[{tag:block, mult:1.0}]` + 本回合承伤减免 |
| 逃跑 | 0 | 敏捷对抗，过则离场 |

### 3.2 技能数据结构（AI 生成只能填这个）
```ts
numeric.combat = {
  cost: 12,                // EP 消耗
  target: 'enemy',         // self|ally|enemy|allEnemy|allAlly|all
  effects: [
    { tag:'deal', mult:1.5 },               // 150% 攻击力档伤害
    { tag:'vulnerable', stacks:2 },          // 目标叠 2 层易伤
    { tag:'strength', stacks:1, turns:99 },  // 自身永久 +1 力量
  ],
}
```
「标签多少、数值多少」就是这一个 `effects` 数组。叙事/外观/品级随便写，战斗只读 effects。

### 3.3 标签注册表（实现见 `systems/combatTags.ts` 的 `TAG_REGISTRY`）

**P0 核心 10 个**
| 标签 | 含义 | 参数 | 结算 |
|---|---|---|---|
| `deal` | 造成伤害 | mult/flat, times | 见 §4 |
| `block` | 获得格挡(护盾) | mult/flat | 加 curShield，回合末清零 |
| `heal` | 回血 | mult/flat | curHp += 量（≤上限）|
| `restore` | 回蓝 | flat | curEp += 量 |
| `strength` | 力量(+伤害) | stacks, turns | 出手伤害 +stacks×攻击档×10% |
| `dexterity` | 敏捷(+格挡) | stacks, turns | 格挡量 +stacks×防御档×10% |
| `vulnerable` | 易伤 | stacks/turns | 目标受伤 ×1.5 |
| `weak` | 虚弱 | stacks/turns | 该单位造伤 ×0.75 |
| `poison` | 中毒(递减) | stacks | 每回合 −stacks HP，然后 stacks−1 |
| `stun` | 眩晕 | turns | 跳过下个行动回合（cannotAct）|

**P1 扩展**：`lifesteal` 吸血 / `thorns` 荆棘反伤 / `regen` 再生 / `burn` 燃烧(定额DoT) / `sunder` 碎甲(防御−) / `silence` 沉默(不能放技能) / `execute` 斩杀(HP<x%秒) / `pierce` 无视格挡 / `cleanse` 净化己方debuff / `dispel` 驱散敌方buff / `taunt` 嘲讽 / `charge` 聚能(下次×N，复用 ChargeState)。
**复用现有结构**：`domain` 领域 → `DomainState`；`charge` → `ChargeState`。

### 3.4 buff/debuff 表达
复用现有 `StatusEffect` + `CombatStatusMod`（playerStore）。两种语义：
- **层数型**（力量/敏捷/中毒/聚能）：数字=强度；有的每回合衰减（毒），有的不衰减（力量）。
- **回合型**（易伤/虚弱/眩晕/燃烧/沉默）：数字=剩余回合，回合末 −1，归零移除。

## 4. 伤害公式（杀戮尖塔同序，必中）
```
base = mult × 出手方有效攻击力档  (或 flat 固定值 / 武器骰)
② × (1 − 虚弱?0.25)         // 出手方虚弱  WEAK_MULT=0.75
③ + 力量层数 × 攻击档 × 10%   // 出手方力量  STR_FRAC=0.10
④ × (1 + 易伤?0.5)          // 目标易伤    VULN_MULT=1.5
⑤ 先扣目标护盾(格挡)，溢出扣 HP
⑥ 暴击(可选)×1.5（默认关）
```
**有效攻击力档**来自 `derivedStats`（patk/matk）+装备+技能树，沿用 `DMG_SCALE` 压平到一场 ~4–6 回合。
**格挡量** = mult×有效防御档 + 敏捷层数×防御档×10%，回合末清零（除非 `barricade`）。

## 5. 描述→标签映射示例（储君·地形改造）
玩家要技能「地形改造」（叙事=用宇宙力量塑造有利地形）。AI 强制映射：
```ts
{ cost:8, target:'self', effects:[ { tag:'strength', stacks:3, turns:99 } ] }
// 叙事 desc: "引动星轨之力，脚下大地隆起为我所用"
```
前端只认 `strength+3` → 之后每次 deal 多 3×攻击档×10%。日志记 `地形改造 → 力量+3`。AI 润色成「他以宇宙之力重塑战场，气运加身，招式愈发凌厉」。**任意花哨描述 → 有限标签 → 前端能算 → AI 还原花哨叙事**。

## 6. 本地敌人 AI（零 API）
借 STS 意图机制 + 简单权重，**确定性**（种子=battleId+round+actorId，保证联机房主/来宾一致）。每回合预告下回合意图（⚔/🛡/✦）。决策优先级：
1. 濒死(HP<25%)且有治疗/逃跑 → 高权重自救
2. 自身无增益且有强化技 → 概率先 buff
3. 目标无 debuff 且有控制/易伤 → 概率上 debuff
4. EP 够 + 大招 CD 好 → 放大招
5. 否则 → 普攻/基础攻击技

权重由敌人定位调（狂战偏攻/法师偏技/辅助偏奶）。整场 0 次 LLM。

## 7. 战斗日志 → 战报 → 一次润色（借 TGv3.2）
结束时压成关键事件一行：
```
BATTLE_RECORD: 结果=胜 | 回合=6 | 我方=[主角:轻伤] | 敌方=[暗影刺客×2:KO] | 关键=[第3回合 易伤+斩杀连斩, 刺客中毒倒地] | 处置=[击杀]
```
AI 提示词（重写 `COMBAT_SUMMARY_RULE`→`COMBAT_NARRATE_RULE`）融合两卡：
- **数值已落库，严禁回复里再改任何数值变量**（TGv3.2 铁则）。
- **叙事沉浸**：正文禁出现回合/层数/易伤/HP 等系统词，转译成伤势/气势/破绽。
- **动作有来有回**：发起→反应→交锋惯性→体力损耗，穿插台词与立场冲突（6.png），禁随意倒地。
- 伤亡以「处置」字段为准（杀/重创/手下留情）。

## 8. AI 生成技能强制套枚举
1. 所有产技能入口（技能树/副职业配方/正文 addSkill/竞技场对手/NPC 生成）加 `SKILL_COMBAT_TAG_RULE`：必须输出 `numeric.combat.effects[]`，tag 只能取注册表枚举，附枚举表+范例。
2. 前端校验兜底：丢弃非法 tag；整块缺失用关键词启发式反推（`inferEffectsFromSkill`），旧档不崩。
3. desc/外观/品级自由，战斗只读 effects。

## 9. 架构与调用图（漏斗）
所有战斗都走：`assembleBattle()` 进 → `settleAction()`/`advanceTurn()` 算 → App 的 `resolveAndNarrate`/`runNpcTurn`/`finishBattle` 驱动。
| 系统 | 接法 | A 替换影响 |
|---|---|---|
| 普通战斗 | CombatSetup → startCombatWithSelection → assembleBattle | 主战场，核心改 |
| 竞技场 | startCombatWithSelection 调用方 + finishBattle 结算 | 签名稳定即几乎不动，回归验证 |
| 联机 raid | startRaidCombat/startRaidDungeon → assembleBattle | 同上，需敌人 AI 确定性 |
| 赌坊角斗场 | casinoEngine.ts 完全独立 | **不碰** |
| 深渊地牢 | abyssEngine.ts 自带 Layer2 沙盒 | **不碰** |

→ 保持 `assembleBattle/settleAction/advanceTurn` 签名稳定，竞技场/raid 仅回归验证，赌坊/深渊不在范围。

## 11. 落地顺序（每步可独立 build）
1. **数据就位**：combatTags.ts + Skill.combat 字段 + inferEffects 兜底（不动引擎）。← 本次
2. **换内核**：重写 settleAction 走标签 VM（单挑）+ 单测绿。
3. **去 API 敌人**：enemyAI.ts 接 runNpcTurn，驱动即时化。
4. **收尾换装**：battleRecord.ts + COMBAT_NARRATE_RULE，改 finishBattle，删两个旧 phase。
5. **面板**：Buff 条 + 意图预告；CombatManager 2 阶段。
6. **技能生成套枚举**：SKILL_COMBAT_TAG_RULE 挂各入口 + 老技能批量兜底。
7. **回归**：竞技场 + 联机 raid + build。

## 12. 风险 / 兼容
- 旧档进行中的战斗：结构变了 → 迁移时 clearCombat（战斗是临时态，无损）。
- CombatPreset 旧 4 字段 → persist 加 version，旧字段安全丢弃。
- 联机：enemyAI 必须确定性（已用 battleId+round 种子）；raid 的 raidRoundTick（燃域/点名/韧性/恐惧）在 App 层、与新引擎正交，保留。
