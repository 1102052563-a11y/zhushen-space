# 部署到网页（让别人也能在线玩）— Cloudflare Pages 指导

本应用是**纯前端 SPA**：没有后端、没有共享数据库。"让别人也能网页玩" = 把构建出的静态文件（`dist/`）放到静态托管上，分享网址即可。和 `fanren-remake.pages.dev`（凡人重制版的线上版本）一模一样的做法。

## 运行模型（每个玩家互相独立）
- **存档**：存在各自浏览器（localStorage / IndexedDB），不互通、不上传 —— 纯单机存档。
- **API Key**：每个玩家自己在 app 设置里填，存自己浏览器，**绝不进构建产物** —— 公开托管不会泄露你的 key。
- **跨域(CORS)**：本地 `npm run dev` 靠 `/dev-proxy` 绕过；线上没有它，所以本仓库自带了一个 **Cloudflare Pages 代理函数**（见下）来解决。

---

## 一、已为你准备好的代理函数

文件：`zhushen-space/zhushen-space/functions/proxy/[[path]].js`

- 它是 Cloudflare Pages 的 serverless 函数，和你的站点**同源**，在服务器端把玩家的 API 请求转发到真正的上游（官方 OpenAI / Gemini 兼容网关 / 各种中转），并把响应（含流式）原样回传。
- 因为是同源转发，浏览器跨域问题被彻底绕开，**玩家可以直接用官方 API 地址，不必再去找"支持跨域的中转站"**。
- 玩家的 Key 只是被透传给上游，**不会存在你这边**。
- 可选防滥用：在 Cloudflare Pages → 设置 → 环境变量 加 `ALLOWED_HOSTS`（逗号分隔，如 `api.openai.com,api.baimeow.icu`），只放行这些上游主机。

## 二、部署步骤

### 方法 A：连 GitHub 自动部署（推荐，最省心，支持自动重建）

1. **在 app 文件夹里初始化 git 并推到 GitHub**（仓库根=项目本身，最省心）：
   ```powershell
   cd C:\Users\Administrator\Desktop\前端卡\files\zhushen-space\zhushen-space
   git init
   git add .
   # 首次用 git 需先设身份（设过就跳过这两行）：
   git config --global user.name "你的名字"
   git config --global user.email "你的邮箱"
   git commit -m "init"
   git branch -M main
   # 先在 GitHub 网站建一个空仓库拿到地址，把下面换成你的：
   git remote add origin https://github.com/你的用户名/zhushen-space.git
   git push -u origin main
   ```
   `.gitignore` 已自动排除 `node_modules`/`dist`，不会把几万个依赖文件传上去。
2. 打开 https://dash.cloudflare.com → **Workers & Pages** → **Create** → **Pages** → **Connect to Git**，选刚才的仓库。
3. 构建设置填：
   - **Framework preset**：None（无）
   - **Build command**：`npx vite build`  ← ⚠️ 不要用 `npm run build`（tsc 会因预存类型错误失败）
   - **Build output directory**：`dist`
   - **Root directory (advanced)**：填 `zhushen-space/zhushen-space`（仓库建在最外层 `files\` 时要指到内层 app 目录；若仓库直接建在 app 文件夹里则留空）
4. 点 **Save and Deploy**。几分钟后拿到网址 `https://你的项目名.pages.dev`。
   - `functions/` 目录会被 Cloudflare **自动识别**并部署成 `/proxy/*` 接口，无需额外配置。
5. 以后改了代码 `git push`，Cloudflare 自动重建。

### 方法 B：Wrangler 命令行直传（不想用 git 时）

```powershell
cd C:\Users\Administrator\Desktop\前端卡\files\zhushen-space\zhushen-space
.\node_modules\.bin\vite build            # 先构建出 dist/
npx wrangler login                         # 首次会弹浏览器授权
npx wrangler pages deploy dist --project-name=zhushen-space
```
- 在**内层项目目录**（含 `functions/` 和 `dist/` 的那层）运行，Wrangler 会自动把 `functions/` 一起打包上传。
- 之后每次更新：重新 `vite build` 再 `wrangler pages deploy dist ...`。

## 三、玩家怎么填 API（部署后告诉玩家）

在 app 里打开 **综合设置 → API 接口库**（或各功能的 API 设置），新增/编辑一个接口：

- **接口地址(baseUrl)**：`https://你的项目名.pages.dev/proxy/<上游地址>`
  - 用官方 OpenAI 兼容端点：`https://你的项目名.pages.dev/proxy/api.openai.com/v1`
  - 用某个中转站：`https://你的项目名.pages.dev/proxy/中转域名/v1`
- **API Key**：玩家自己的 key
- **模型(modelId)**：对应上游支持的模型名

> app 会自动在 baseUrl 后面接 `/chat/completions`，所以 baseUrl 只填到 `/v1`（或上游要求的前缀）即可。

## 四、注意事项

- **base 路径**：根域名部署（`xxx.pages.dev` 或自定义域名根）不用动配置。若你硬要放在子路径下，需在 `vite.config.ts` 加 `base: '/子路径/'` 再重新构建，否则白屏。
- **dev-proxy 不影响线上**：`vite.config.ts` 里的 `/dev-proxy` 只在本地开发用，构建产物里没有它，可以不管。
- **大字体**：`public/fonts/msvista.woff2` 约 3.8MB，首次加载稍慢但会被浏览器缓存，可接受。
- **自定义域名**：Cloudflare Pages 项目 → Custom domains，可绑你自己的域名。
- **想让玩家"连地址都不用填"**：可以把 app 默认 API 地址预填成代理形式（相对路径 `/proxy/api.openai.com/v1` 即指向同源代理），玩家只需填 Key。需要的话告诉我，我帮你改默认值（会锁定一个默认上游）。
