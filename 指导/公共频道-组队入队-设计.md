# 公共频道 · 组队入队 设计（待实现）

> 记录于设计讨论阶段，**尚未实现**。核心结论：频道组队 = **临时世界队伍**，与已有的「永久冒险团」分开两层。

---

## 一、核心区分：临时世界队伍 ≠ 永久冒险团

| | 🤝 临时世界队伍（频道组队，本设计） | 🛡 永久冒险团（已实现，`adventureTeamStore`） |
|---|---|---|
| 范围 | **只在当前任务世界/副本有效** | 跨世界永久保留 |
| 来源 | 公共频道「组队帖」一键入队 | 剧情明确「建立冒险团」 |
| 成员 | 临时队友（NPC，带"当前世界"标记） | 正式团员（`TeamMember`，C-id） |
| 离队 | **世界结束自动解散** / 剧情或手动 | 剧情 / 考核试炼 |
| 晋级 | 无 | 阶位 E→SSS、经验+活跃度双计量 |

理由：频道里的契约者是萍水相逢、一起刷个本就散，做成"临时队伍"最自然，不污染冒险团的晋级体系。后续可在世界结束时把表现好的临时队友「转正」输送进冒险团，打通两层。

---

## 二、数据落地：复用 NPC 系统（不新建 store）

给 `NpcRecord`（`npcStore`）增加三个字段：
- `partyMember?: boolean` —— 是否当前的临时队友
- `partyWorld?: string` —— 为哪个世界（worldName）入队的
- `partyRole?: string` —— 队内职责（坦克/治疗/输出/侦察…）

临时队友天然拥有六维/装备/技能/在场离场，与现有 NPC 面板、在场浮窗无缝。在场浮窗 / NPC 列表给队友打一个「队」徽章。

（可选）也可在 `adventureTeamStore` 之外加一个极轻的 `partyName` 显示队名，或直接不要队名。

---

## 三、入队逻辑（确定性，类比交易的「购买」）

组队帖（`ChannelRecruit`：role/targetWorld/reqTier/slots/reward）加一个「**加入/组队**」按钮，点击后**代码确定性处理**（不经 AI 报数）：
1. 为该发帖契约者**建/取一个 NPC 档案**（姓名/阶位/标签/职责从帖子带过来）；
2. 标记临时队友：`partyMember=true`、`partyWorld=当前 worldName`、`partyRole=…`；
3. 设为**在场**（`onScene=true`）；
4. 帖子标记"已组队"（按钮变灰，类比交易帖的 `traded`）。

- 不走货币；帖子的 `reward`（酬劳/分成）记进档案备注。
- `reqTier`（阶位门槛）只做**软提示**，不硬卡。
- 玩家是主角，"加入别人的队"与"招募别人入队"在实现上统一为：发帖人成为玩家身边的临时队友 NPC。

---

## 四、离队逻辑（重点）

两种触发，**都不用脆弱的关键字匹配**：

### ① 世界结束 → 自动解散（确定性生命周期）
- 在 `enterWorld`（切到新世界）与 `reconcileHomeWorld`（回归主神空间）时：
  把所有 `partyMember && partyWorld ≠ 新世界` 的临时队友自动离队——
  `partyMember=false` + **离场归档**（`removeNpc` 软删除，保留 NPC 档案，**不物理删除**）。
- 绑在"世界切换"这个明确动作上，不会误判（区别于 NPC `isDead` / 势力 `isDestroyed` 那种关键字踩雷）。

### ② 世界中途离队 → 正文 / 手动
- 正文明确写队友牺牲/翻脸/中途离开 → AI 指令 `partyLeave("C1")`；
- 或队伍面板上手动「踢出 / 离队」按钮。

---

## 五、可选增强：离队时「转正」进冒险团
世界结束自动解散时，对某个想长期留用的临时队友，弹「**邀请加入冒险团**」：
把他从临时队友转进 `adventureTeamStore.members`（永久）。不转的正常离场归档。
→ 临时队伍给永久冒险团**输送人才**，两层打通。

## 七、涉及改动（预估）
- `npcStore`：加 `partyMember/partyWorld/partyRole` 字段 + `joinParty/leaveParty` actions + 世界切换清理。
- `App.enterWorld` / `reconcileHomeWorld`：世界结束自动解散临时队伍。

- （可选）队伍面板 / 在场浮窗「队」徽章 / 转正询问弹窗。
