# 条目编写规范

本页定义所有条目的统一格式，确保逐章扩充时风格一致、可交叉引用、不重复、不剧透。

## 总原则
1. **忠于原著**：只写正文明确写到的内容；推断须标注「（推测）」。
2. **渐进不剧透**：只记录已读章节的信息；未读到的不提前写。
3. **增量更新**：同一对象再次出场，**在其已有页面追加**，不新建重复页。
4. **必标出处**：关键信息注明相关章节，如 `（第123章）`。
5. **命名**：以原著正式名作页面名；别名写进 front-matter `别名`。
6. **关系（必备）**：每个人物条目都应含「**关系**」一节，列出其与**其他人物**的关系，并用相对链接互链（NPC ↔ 主角、NPC ↔ NPC）；对方有专页则链页、无专页则纯文本注明。

## front-matter 模板

### 人物
```yaml
title: <姓名>
分类: 人物
身份: 契约者 / NPC / 反派
阶位:
天赋:
所属:
别名: []
首次登场: 第X章
状态: 存活 / 阵亡 / 未知
```
正文建议分节：**简介 / 外貌 / 性格 / 能力（天赋·技能·装备）/ 经历（按章节）/ 关系 / 出处**。

### 任务世界
```yaml
title: <世界名>
分类: 任务世界
难度:
人数: 单人 / 组队
```
正文：**背景 / 任务目标（主线·支线·隐藏）/ 关键NPC与势力 / 奖励与惩罚 / 出场契约者 / 出处**。

### 势力
```yaml
title: <势力名>
分类: 势力
类型: 冒险团 / 组织 / 国家 / 宗门
所属世界: 轮回乐园 / <任务世界>
```

### 技能·功法 / 装备道具 / 物品
```yaml
title: <名称>
分类: 技能 / 装备 / 消耗品 / 材料
品级:
来源: 乐园商店 / 任务掉落 / 自创
```

### 体系页（阶位 / 天赋等框架）
用表格列层级，细节随章节补；断言框架须加 `!!! tip "来源"` 说明。

## 主角 / 常驻角色（会随剧情成长）
苏晓这类持续成长、跨多个世界的角色，按以下规则维护，避免「覆盖即丢史」或页面变一坨：

1. **当前值覆盖，历史只增不删**：六维 / 等级 / 法力 / 乐园币 / 当前装备等「现状」覆盖到最新；每次变动同时往「**成长轨迹**」表**追加一行**（章节 ｜ 变动 ｜ 来源）。
2. **区分两层**：
    - **持久**（跨世界带走）：基础六维、等级 / 阶位、天赋、法力、乐园币、规则物品、储物空间、已解锁功能。
    - **世界限定**（留在该世界）：不可带出的装备、本地货币、临时身份 / 化名、任务世界 NPC 关系、世界之源。
3. **两个聚合视角**：
    - **角色页 = 纵向（时间）**：简介 / 性格 → 当前状态（快照）→ 成长轨迹（表）→ 历程（按世界分块，每块精简 + 链到世界页）→ 持久关系。
    - **任务世界页 = 横向（空间）**：该世界的人物 / 势力 / 地点 +「角色在本世界」（收获 / 世界之源 / 进度）+ 世界结束时的**结算快照**：① 世界结算（世界之源 / 任务数 / 综合评价 / 奖励）；② **苏晓离世当下的定格**——等级 / 职业 / 天赋 / 技能 / 六维 / 法力 / 乐园币 / 属性点 + **当前装备（离世时穿戴：武器 / 防具 / 饰品，逐件列出、未穿戴的归入携出物）** + 携出物与留下物；③ **随从 / 召唤物快照**——若苏晓名下有随从 / 契约生物 / 召唤物，逐个记其离世定格（名称 / 来源 / 等级或属性 / 能力 / 去向：带出·留下·阵亡·解约），详情归 [苏晓的随从与召唤物](../人物/苏晓的随从与召唤物.md) 页、世界页只摘要 + 链接。**须注明此为离世瞬间、与角色页「当前状态」不同**（如海贼王离世时尚无灭法之影职业）。
        - **随从 / 召唤物专页**：苏晓获得任何随从 / 契约生物 / 召唤物，都在 [苏晓的随从与召唤物](../人物/苏晓的随从与召唤物.md) 单列一节（来源 / 能力 / 归属 / 历程），并在每个世界结束时为在册随从补结算快照（与苏晓本体分开记，避免淹没主页）。
4. **关系只有一个正主**：世界 NPC 的详情写在该 NPC 页，角色页 / 世界页只链接、不复制。

## 交叉引用
用相对链接互链（指向 `.md` 文件），例如 `[苏晓](../人物/苏晓.md)`、`[阶位](../力量体系/阶位.md)`。
