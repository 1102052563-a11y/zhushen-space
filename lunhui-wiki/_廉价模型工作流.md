# 廉价模型工作流（轮回乐园百科 · 学徒批次作业手册）

> 本文是「用廉价模型继续建百科」的完整作业规程。**学徒 = 廉价模型**（干批量读写），**师傅 = Claude**（闸口：nav / build / QA / 进度），**你 = 发工单、传话、拍板**。
> 学徒的三份必读（每批开工都读，缺一不许动笔）：
> ① `_必读须知.md`（§0 进度指针 + §1 铁则 + §2 流程）② `docs/参考/编写规范.md`（模板与快照铁律）③ 本文（分工、边界、**内容深度标准**、交接格式）。

## 一、角色分工与禁区

| 角色 | 做什么 | 绝对禁区 |
|---|---|---|
| **学徒（廉价模型）** | 读本批原文 → 抽实体 → 新建/增量修改 `docs/**/*.md` → 写交接单 | **禁改**：`mkdocs.yml`、`PROGRESS.md`、`_必读须知.md`、本文、`scripts/`、`_source/`、`site/`；**禁跑**：build/serve/pip/git，任何命令（只许 Grep 定位 + Read 读文件 + 写 docs 下的 md 与 `_交接/` 交接单）；**禁用**：自己训练记忆里的《轮回乐园》知识与任何外部资料（只准写本批原文 + 既有 wiki 页里有的） |
| **师傅（Claude）** | 收交接单 → 跑机检 `scripts/check-wiki.py <本批文件>` → 把新页合入 `mkdocs.yml` nav → `mkdocs build --strict` → QA 抽查 → 更新 `_必读须知 §0` + `PROGRESS.md` | — |
| **你** | 把「工单」发给学徒；学徒交批后叫师傅收尾；对存疑项拍板 | — |

> 为什么 nav 不让学徒碰：`mkdocs.yml` 97KB、人物按所属世界分组有严格次序（曾放错组返工），廉价模型改 YAML 风险太高。学徒只在交接单里**申报**「新页 → 应挂哪组」。

## 二、内容深度标准（⚠ 核心 · 写薄了=返工）

百科的价值在**细**：谁、在哪、因为什么、做了什么、结果如何、数值变了多少。一句话带过=白干。

### 2.1 人物分级与字数配额（正文去空白字数，不含 front-matter）

写页前先给人物定级（判据写进交接单）：

| 级 | 判据 | 新建页配额 | 必备小节 |
|---|---|---|---|
| **S** | 苏晓本体、随从/召唤物（布布汪等） | 已有专页，只增量：本批每次数值/装备/技能/状态变动都进「成长轨迹」表 + 历程新增小节 ≥300字/批 | （已有结构） |
| **A** | 本批戏份重：跨多章出场 + 有能力展示 + 与苏晓直接互动（如 boss、盟友、重要商人） | **≥800 字** | 简介 / 外貌·性格 / 能力（天赋·技能·装备**逐条**）/ 经历（**按卷章分段**）/ 关系（逐人一行）/ 出处 |
| **B** | 有名有戏份但出场少（单章、一两场戏） | **≥400 字** | 简介 / 能力 / 经历 / 关系 / 出处 |
| **C** | 仅被提及、一闪而过、当场领便当 | **≥150 字**，可注「（信息有限·N卷N章仅提及）」 | 简介 / 出处（能写则加关系） |

- 配额是**下限**；原文信息多就往上写，不许为省事压到线上。
- 群像/配角合集页（多个无独立戏份小角色合并）例外，但**每人仍要 2~4 句**：身份+做了什么+下场。

### 2.2 各类页面深度要求

- **经历节（人物页的重头）**：按卷章分小节（`#### 48卷37-41章`），每段必含五要素——**时间地点 / 事件因果（为什么发生）/ 过程关键动作 / 数值·装备·状态变化（原样抄原文数字）/ 结果与影响**。关键对话用转述+短引（一两句原话点睛即可，不整段抄）。
- **能力节**：天赋/技能**逐条列**（名称+效果+品级/等级+出处章），装备**逐件列**（名称+能力+来历+现归属）。不许写"实力强大""身手不凡"这种空话。
- **数值铁则**：六维/等级/法力/乐园币/世界之源%/伤害数字——**一律照抄原文**，不四舍五入、不脑补、原文没写就写「不详」。
- **世界页（进行中的世界）**：每批往「主线进程」节追加本批进展 **150~400 字**（含：推进了什么、新势力/地点、苏晓当前处境），新出现的势力/地点/机制在对应小节展开。
- **时间线**（`docs/总览/时间线.md`）：每章 **1~3 行**，格式 `- **48卷37章**：<事件>（<结果/意义>）`。
- **章节梗概**（`docs/参考/章节梗概` 对应卷页）：每章一段 **150~300 字**，起因→经过→结果，含出场人物与数值变化要点。**不是一句话**。
- **装备/技能/物品页**：来历（谁给的/哪掉的/多少钱）+ 能力全文（原文效果描述照录要点）+ 品级 + 使用/成长记录（按章追加）。

### 2.3 合格 vs 不合格对照（写前必看）

**❌ 不合格（经历段——一句话糊弄）：**
> 48卷中，凯撒加入了深渊滋生物阵营并帮了苏晓一次。

**✅ 合格（同一事件·五要素齐全）：**
> **48卷24-27章·深渊滋生物阵营的"代获"操作**：凯撒潜入深渊滋生物阵营，冒名顶替一名 Lv.MAX+恶霸 Lv.EX 的存在；期间一笔本属他人的 Lv.EX 击杀奖励，被他动手脚改判为由苏晓代为获得——代价是自己被扣信誉度 170。此举延续两人"互坑又互救"的关系基调，也为苏晓在幽暗大陆的资源积累添了一笔外快。（48卷24-27章）

**❌ 不合格（能力节）：**
> 实力强大，拥有多种天赋和技能。

**✅ 合格：**
> - 天赋·猎影（灭法第三天赋）：48卷33-36章在大教堂祭祀坛以 31% 黑暗之源提升，天然完成一次觉醒、只差二次觉醒（每次消耗 5%、每种天赋最多提升 4 次）。
> - 武器·银月之刃：本卷成长两次（48卷33-36章）。

**❌ 不合格（章节梗概）：**
> 苏晓打败了钢铁使徒。

**✅ 合格：**
> **48卷33-36章**：苏晓借凯撒的恶浊臭鞋逼出公爵队，鹿格被熏昏、凯因化暗魂骷髅逃走、雪怪被罪亚斯斩杀，苏晓一刀贯穿钢铁使徒核心。真相反转：公爵早被苏晓下了软化金属的炼金合成物、成了死局——他故意植入钢铁使徒核心唤醒其占据自己旧躯送死，自己则夺占克兰克身躯（内封克兰克/克萝/公爵之妻三道灵魂，想"一家团聚"）。苏晓获 11% 世界之源、机械核心（半损）、罪人徽章。

## 三、批次循环（学徒执行 · 每批 3~5 章；人物密集的批取 3 章）

### 步0 · 开工闸（不做完不许动笔）
1. Read `_必读须知.md` §0~§2、`docs/参考/编写规范.md`、本文全文。
2. 在回复开头**复述**：a) 当前进度指针（卷·章）b) 本批要读的章节范围 c) 五条铁则（忠于原著 / 渐进不剧透 / 增量更新 / 必标出处 / 一个事实一个正主）d) 人物 A/B/C 三级字数配额。复述不出来就重读。

### 步1 · 取原文（只读本批，越界=违规）
原文：`_source/轮回乐园-utf8.txt`（46MB，**严禁整读**）。格式：卷标题 `第四十八卷 幽暗大陆·死寂之根源`，卷内章标题 `第一章：进入`（中文数字，**每卷重新编号**，每章约 170 行）。
定位方法：
1. Grep 卷标题 `^第.{1,6}卷` 拿到当前卷起始行；
2. Grep 章标题 `^第.{1,6}章` 拿到本批起章与「批末下一章」的行号；
3. Read 只读 `[起章行, 下一章行)` 区间（用 offset/limit 分段读，每次 ≤300 行）。
**红线**：读到批末下一章的标题行就停。多读一章都算剧透抢跑。

### 步2 · 实体清点（先规划后动笔·清单贴进回复）
通读本批原文后，先输出「实体清单」再改任何文件，格式：

```
## 批次实体清单（48卷37-41章）
### 🆕 新具名人物（逐个定级）
- <名>（A级·37-40章·钢铁使徒之师·与苏晓交易）→ 新建 docs/人物/<名>.md
- <名>（C级·39章仅提及）→ 新建 docs/人物/<名>.md
### ♻️ 旧人物变化
- 苏晓：<列出全部数值/装备/技能/状态变动，带章节>
- <名>：<变动>（若死亡 → front-matter 状态改「已死亡（48卷N章·死因）」）
### ⚔️ 装备/技能/物品 · 🏛 势力 · 📍 地点 · 🧬 生物万族 · ⚙️ 新机制
- <逐条：名称+一句话+落到哪个页>
### 🏁 通关信号：有/无（有 → 步4 必须执行）
### 📝 时间线/梗概：37~41 章各 1~3 行 / 各一段
```

### 步3 · 逐实体落页（增量，不覆盖历史）
- **新人物** → 按编写规范 front-matter 模板单开 `docs/人物/<名>.md`，按 §2.1 配额与小节写**满**；殁者标「（殁·N卷N章）」。**绝不只在 index.md 加一条 bullet**。
- **旧实体** → 先 Read 其现有页，再**追加**新信息：人物页在「经历」下加本批卷章小节（五要素）；不改旧段落（除非旧信息被原文推翻，此时改动+注明「N卷N章修正」）。
- **苏晓页** → 现状（六维/装备/币…）**覆盖**到最新，同时往「成长轨迹」表**追加行**（章节｜变动｜来源），一个变动一行不合并；历程按世界分块，本批所在世界块追加叙述。
- **随从/召唤物**（布布汪、咕噜等）→ 各自专页同规格更新。
- `docs/总览/时间线.md` 与 `docs/参考/章节梗概` 对应卷页按 §2.2 配额追加。
- **出处一律带卷**：写 `（48卷37章）`，不写裸「第37章」（每卷章号重开，裸章号歧义）。
- **排版铁则**：正文段落与紧随的 `- ` 列表之间**必须空一行**；内部链接用相对 `.md` 路径；文件 UTF-8。

### 步4 · 通关快照（头号铁律 · 漏了=返工）
本批原文出现任何世界通关/离世（世界结算、【世界已结束】、传送回乐园、世界并入）→ **当批立刻**在该**世界页**（`docs/世界/任务世界/<世界>.md`）`## 出处` 之前补「🏁 通关结算快照」，**版式逐字对齐模板页 `fate-zero.md` / `火影忍者.md` / `海贼王.md`**：

```
## 🏁 通关结算快照（X卷X章 · 离世瞬间，与当前状态不同）
### ① 世界结算            ← | 项 | 值 | 表格（难度·获得世界之源·完成任务·综合评价·奖励·停留时长）
### ② 离世定格（…那一刻）  ← | 项 | 值 | 表格（等级/阶位·六维〔独立列全六项+有效值〕·法力·核心技能·副职业·
                              当前装备〔武器/刀鞘/附武器/项链/戒指/手部/上衣/脚部/装饰/探测…逐件列全〕·
                              携出物·留在本世界·善后）
### ③ 随从快照            ← 无序列表（每随从：形态/品级·属性·技能·本世界作用·去向〔带出/留下/阵亡/解约〕）
```

- **绝不用 `!!! success/note` 等 admonition 代替表格**（踩过坑被要求返工）。
- ② 用「世界结束那一刻」的数值；返园后的强化**不计入定格**，只在注里说明。装备逐件以苏晓页当前装备账为准。
- 同一世界多次进入 → 每次独立快照/独立页，绝不混写。苏晓页只留一句链接指向世界页快照。
- 无主线的特殊世界（晋升任务/独立试炼）照补，特殊栏写「—」。
- 交接单里把世界页标 `✅通关`，师傅改 nav 状态。

### 步5 · 自检清单（逐项勾完才交批）
- [ ] 本批每个新具名人物都有独立页；A≥800 / B≥400 / C≥150 字**逐页达标**（去空白数）
- [ ] 人物页小节齐全（简介/能力/经历/关系/出处），front-matter 含 title/分类/所属世界/身份/状态
- [ ] 经历段五要素齐全（时间地点/因果/动作/数值变化/结果），无"一句话经历"
- [ ] 所有数值照抄原文；所有关键信息标「N卷N章」；没有任何超出本批进度的内容
- [ ] 旧页是追加不是覆盖；苏晓成长轨迹逐变动加行
- [ ] 时间线每章 1~3 行；章节梗概每章 150~300 字
- [ ] 段落与列表之间有空行；内部链接是相对 `.md`
- [ ] 有通关 → 快照三段式表格已补在世界页
- [ ] 没有动过禁区文件

### 步6 · 写交接单
写到 `_交接/<卷>卷<起章>-<末章>.md`（如 `_交接/48卷37-41.md`），格式：

```markdown
# 批次交接单 · 48卷37-41章
## 1. 本批范围
卷·章 + 原文行号区间
## 2. 改动文件与字数
### 新建：
| 页面 | 级别 | 正文字数 | 一句话内容 |
|---|---|---|---|
| docs/人物/xxx.md | A | 923 | 钢铁使徒之师·与苏晓的交易 |
### 修改：
| 页面 | 改了什么（一句话） |
|---|---|
## 3. nav 挂载申报
新页路径 → 应挂分组（人物按 front-matter 所属世界；任务世界按剧情顺序；注明插在谁后面）
## 4. 通关事件
无 / 有 →（世界页路径 + 快照已补 + 待改 ✅通关状态）
## 5. §0 进度指针草稿
照 _必读须知 §0 现有格式，续写本批一段（师傅审后合入，学徒不直接改 §0）
## 6. 存疑与遗留
原文没写清的、归类拿不准的、发现的既有页错误
```

### 步7 · 师傅收尾（学徒到步6为止）
`python scripts/check-wiki.py <本批全部 md>`（指定文件模式会额外查**内容深度**）→ nav 合入 → `python -m mkdocs build --strict -f lunhui-wiki/mkdocs.yml` → QA 抽查 → 合入 §0 + PROGRESS.md → 提交。

## 四、师傅 QA 抽查表（每批）

1. **深度**：对照交接单字数表抽 2 页数字数；抽 1 个 A 级人物页看经历五要素、能力逐条、关系逐人。
2. **快照**：有通关的批 → 世界页快照三段式表格是否逐字对齐模板页；装备是否逐件；定格数值是否"离世那一刻"。
3. **人物 nav 分组**：新人物页 front-matter `所属世界` 与申报分组一致（踩过坑：7 名魔禁人物挂错恶魔古堡组）。
4. **抢跑**：机检的越进度引用报告 + 随机抽 2 页核对出处章节确实在已读范围。
5. **增量**：抽 1 个旧实体页 git diff，确认是追加不是重写/删史。
6. **数值**：抽 3 个数字回原文核对（六维/%/币）。
7. **格式**：build --strict 零死链（Material 团队推广 Warning 不是错误，见 `Documentation built in` 即成功）。

## 五、常见翻车对照表（全部真实踩过/高危）

| 翻车 | 正确做法 |
|---|---|
| **内容写薄：一句话经历、"实力强大"式能力** | §2 配额+五要素+逐条列，写满为止 |
| 用 admonition 写结算快照 | 三段式 `##/###` + 表格，模板=火影/海贼页 |
| 新人物只在 index.md 加 bullet | 每个具名人物单开页 + nav 申报 |
| 人物 nav 挂错世界组 | 以 front-matter `所属世界` 为准申报分组 |
| 段落紧贴列表不空行 | 之间必空一行 |
| 用训练记忆/网上资料补设定 | 只写本批原文 + 既有页内容，未读的不写 |
| 覆盖苏晓页丢历史 | 现状覆盖 + 成长轨迹只增不删 |
| 数值凭印象写/四舍五入 | 一律照抄原文，没写就「不详」 |
| 裸「第N章」出处 | 一律「N卷N章」 |
| 快照混入返园后强化数值 | 定格=离世那一刻，返园强化只写注 |
| 读原文越过本批末章 | 到批末下一章标题行即停 |
