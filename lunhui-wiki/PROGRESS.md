## 技术栈
- **Material for MkDocs**（Python），与凡人百科同款；中文搜索靠 `jieba` 分词。

- 本地预览：`python -m mkdocs serve -f lunhui-wiki/mkdocs.yml`（默认 http://127.0.0.1:8000）
- 构建（严格模式，死链即报错）：`python -m mkdocs build --strict -f lunhui-wiki/mkdocs.yml`

## 分类演进原则（大类会随阅读生长）
当前骨架是**临时假设**（据既有认知 + 凡人模板搭的），每批读完复核一次：
- **默认**：新内容进现有大类、当条目或子页加——多数批次不动大类。
- **加 / 拆大类**：只当出现一类**反复出现、且有自身结构**的新事物、硬塞进现有桶会伤导航时才做。可预见的例子：「万族」若膨胀 → 从 世界/生物万族 提为独立大类；道具若爆量 → 拆「本命武器 / 契约武器」；乐园机制若展开 → 商店 / 兑换 / 裁决 / 会员各成页；后期本源 / 规则 / 气运 → 把「纪元与宇宙背景」扩成多页。
- **加法优先、勿破坏**：宁可新增或拆分，避免重命名 / 移动已有路径（会断链、断 URL）；必须重构时同步改侧栏 + 交叉链接，记入变更记录。
- **暂存桶**：一时归不进去的先记到「待归类」，攒够同类再提为正式分类——别过早建大类，也别丢数据。
- 条目增长**自动**进行；大类调整我会**带一句理由先告知再动手**（结构是设计决策）。
