# 叙事记忆与总结（内容处理）集成指导

把 **fanren-remake** 「内容处理」里的**叙事记忆 + 总结**机制移植到 **zhushen-space**——让攒下来的总结**真正回灌进正文生成**，当长期记忆用。

> 背景：你已经做了「杂项演化」的 `addSmallSummary/addLargeSummary`，但它们目前只是**记录+展示**，没进正文提示词。这份文档解决的就是「怎么把总结塞回生成上下文」这一步。

---

## 0. 一句话

把对话历史按「离现在的远近」分三档喂给模型：**最近 N 楼=原文**、**中段=小总结**、**更早=大总结**，并把被压缩的那些打包成一个 `<过往记忆>…</过往记忆>` 系统消息放在最前。这样上下文不爆，模型又记得住前因后果。

---

## 1. fanren-remake 怎么做的（源码定位）

### 1.1 总结存哪
文件 `assets/runtimePersistence-DI-l3NQM.js` —— **每个回合（楼层）记录**上挂两个可选字段：

```js
// generatedHistory 每条 turn 的 schema（节选）
{ content: string, smallSummary?: string, largeSummary?: string, ... }
```

即 **`smallSummary`/`largeSummary` 是逐楼层附带的**，不是一个全局列表。小总结=该楼层的浓缩，大总结=阶段性浓缩。

### 1.2 三档分类 + 取值（`assets/uiStore-Bda3MKAg.js`）

```js
// 按"距当前的楼层数 e"（0=最新）分档
function jL(e, t){ return e < t.recentFullTextCount ? 'full'
                  : e < t.recentFullTextCount + t.smallSummaryWindowSize ? 'small'
                  : 'large'; }
// 最近 recentFullTextCount 楼永远 full；其余在启用时才按 jL 分档，否则丢弃
function VL(e, t){ return e < t.recentFullTextCount ? 'full' : (t.enabled ? jL(e, t) : null); }
// 按档取内容：full→原文；small→小总结(缺则原文)；large→大总结(缺则小总结/原文)
function BL(turn, type){ return type==='full' ? turn.content
                        : type==='small' ? (turn.smallSummary ?? turn.content)
                        : (turn.largeSummary ?? turn.smallSummary ?? turn.content); }
```

### 1.3 组装进 prompt（同文件 `buildMessages`）

```js
// 伪代码还原
const past = [];                 // 被压缩的（small/large）
const out  = [];                 // 正常 role 消息（full）
if (deepCfg.enabled && deepSummaryText) past.push(`[深度记忆档案]:\n${deepSummaryText}`);
const list = history.filter(IL); // 有效楼层
for (let i = 0; i < list.length; i++) {
  const turn = list[i];
  const type = VL(list.length-1-i, cfg);   // 距当前的档位
  if (!type) continue;
  const content = strip(BL(turn, type)).trim();
  if (!content) continue;
  if (cfg.wrapPastMemory && type !== 'full') past.push(`[过往记忆]: ${content}`);
  else out.push({ role: turn.role, content });
}
if (past.length) out.unshift({ role:'system',
  content:`<过往记忆>\n${past.join('\n\n')}\n</过往记忆>` });   // 整块塞最前
return out;
```

**要点**
- 最近 `recentFullTextCount` 楼 = 原文，照常以各自 role 进消息列表。
- 再往前 `smallSummaryWindowSize` 楼 = 小总结；更早 = 大总结。
- 所有被压缩的（small/large）**合并成一条 `<过往记忆>` system 消息**，连同 `[深度记忆档案]`（=生平/deep summary）一起 unshift 到最前。
- 关闭总结时（`enabled=false`）：只保留最近 `recentFullTextCount` 楼，更早的**直接丢弃**（和 zhushen-space 现在 `historyLimit` 切片行为一致）。

### 1.4 配置（与「生平压缩」共用一套，见 `store-glxxBUjZ.js`）
`recentFullTextCount`(默认 5) / `smallSummaryWindowSize`(默认 25) / `wrapPastMemory`(true) / `enabled`，外加深度总结 `deepSummaryText`（生平档）。

## 2. zhushen-space 现状与差距

| 维度 | fanren-remake | zhushen-space 现状 | 差距 |
|---|---|---|---|
| 历史裁切 | 三档（full/small/large）| `historyLimit` 一刀切，超出**直接丢**（`App.tsx:1414` `messages.slice(-historyLimit)`）| 无中长期记忆 |
| 总结存储 | 逐楼层 `smallSummary/largeSummary` | 杂项演化存**全局列表** `miscStore.smallSummaries/largeSummaries`，**不挂楼层** | 需要可定位到楼层 |
| 总结注入 | `<过往记忆>` system 块 + `[深度记忆档案]` | **完全没注入** | 本指导核心 |
| 配置 | summaryConfig | 无（仅 `historyLimit`）| 需加配置 |

**结论**：要让 `messages`（楼层）能带小/大总结，并在 `callApi`（`App.tsx:1400`）拼 prompt 时按三档组装 + 注入 `<过往记忆>`。

## 3. 落地方案

### 步骤 ① 给楼层挂总结字段

`src/types.ts` 的 `ChatMessage`（或消息类型）加两字段：

```ts
export interface ChatMessage {
  role: 'user' | 'assistant';
  content: string;
  // …原有字段…
  smallSummary?: string;   // 该楼层小总结
  largeSummary?: string;   // 该楼层大总结（阶段性）
}
```

### 步骤 ② 总结从哪来（两选一）

- **方案 A（推荐·复用杂项演化，0 额外调用）**：杂项演化阶段本来每回合就产出 `addSmallSummary`。改成把它**写到刚生成的那条 assistant 楼层**上，而不是只进全局列表。
  - 在 `miscParser` 的 `pushSmall` 旁加一个回调，或在 `runMiscEvolutionPhase` 拿到 small/large 后调 `attachSummaryToLastAssistant(small, large)`：
    ```ts
    function attachSummaryToLastAssistant(small?: string, large?: string) {
      setMessages((ms) => {
        const i = [...ms].reverse().findIndex((m) => m.role === 'assistant');
        if (i < 0) return ms;
        const idx = ms.length - 1 - i;
        const next = [...ms];
        next[idx] = { ...next[idx], ...(small ? { smallSummary: small } : {}), ...(large ? { largeSummary: large } : {}) };
        return next;
      });
    }
    ```
- **方案 B（独立小总结 pass）**：每条正文生成后单独发一次便宜的「请用 1-2 句概括本楼」调用，写回该楼层。更可控但多一次 API。

> 推荐 A：你已经在跑杂项演化，顺手把它的小/大总结落到楼层即可，不额外花钱。

### 步骤 ③ 配置（`settingsStore.ts` 加一段）

```ts
narrativeMemory: {
  enabled: boolean;            // 默认 false
  recentFullTextCount: number; // 最近几楼用原文，默认 6
  smallSummaryWindowSize: number; // 再往前几楼用小总结，默认 20
  wrapPastMemory: boolean;     // 压缩楼层包进 <过往记忆>，默认 true
}
```
配套 setter；merge 回落默认值。可在「正文生成」设置页或杂项演化页加一组控件。

### 步骤 ④ 三档构建器（新 `src/systems/narrativeMemory.ts`）

```ts
import type { ChatMessage } from '../types';

export interface NarrativeMemCfg {
  enabled: boolean; recentFullTextCount: number; smallSummaryWindowSize: number; wrapPastMemory: boolean;
}
type Tier = 'full' | 'small' | 'large' | null;

function classify(idxFromEnd: number, c: NarrativeMemCfg): Tier {
  if (idxFromEnd < c.recentFullTextCount) return 'full';
  if (!c.enabled) return null;                 // 不启用：旧楼直接丢（同现状）
  if (idxFromEnd < c.recentFullTextCount + c.smallSummaryWindowSize) return 'small';
  return 'large';
}
function pick(m: ChatMessage, t: Tier): string {
  if (t === 'full')  return m.content;
  if (t === 'small') return m.smallSummary ?? m.content;
  return m.largeSummary ?? m.smallSummary ?? m.content;   // large 回退链
}

/** 返回要塞进 messages 的「记忆系统消息」+「最近原文消息」 */
export function buildNarrativeHistory(
  history: ChatMessage[], cfg: NarrativeMemCfg, deepSummaryText = '',
): { memory: { role: 'system'; content: string }[]; recent: { role: 'user'|'assistant'; content: string }[] } {
  const past: string[] = [];
  const recent: { role: 'user'|'assistant'; content: string }[] = [];
  if (deepSummaryText.trim()) past.push(`[深度记忆档案]:\n${deepSummaryText.trim()}`);
  for (let i = 0; i < history.length; i++) {
    const m = history[i];
    const t = classify(history.length - 1 - i, cfg);
    if (!t) continue;
    const content = pick(m, t).trim();
    if (!content) continue;
    if (cfg.wrapPastMemory && t !== 'full') past.push(`[过往记忆]: ${content}`);
    else recent.push({ role: m.role, content });
  }
  const memory = past.length
    ? [{ role: 'system' as const, content: `<过往记忆>\n${past.join('\n\n')}\n</过往记忆>` }]
    : [];
  return { memory, recent };
}
```

### 步骤 ⑤ 接进正文生成（`App.tsx` `callApi`）

把现在的「`slice(-historyLimit)` + map」换成三档构建：

```ts
// 旧：
// const visibleHistory = historyLimit > 0 ? allHistory.slice(-historyLimit) : allHistory;
// const history = [...examples, ...visibleHistory.map(...), { role:'user', content:userText }];

// 新：
const cfg = useSettings.getState().narrativeMemory;
const deep = ''; // 可选：接 miscStore 全部大总结 join('\n') 或 生平/memoryStore 长期档
const { memory, recent } = cfg.enabled
  ? buildNarrativeHistory(allHistory, cfg, deep)
  : { memory: [], recent: (historyLimit > 0 ? allHistory.slice(-historyLimit) : allHistory)
        .map((m) => ({ role: m.role as 'user'|'assistant', content: m.content })) };

const history = [...examples, ...memory, ...recent, { role: 'user' as const, content: userText }];
// reqBody.messages = [{ role:'system', content: sysPrompt }, ...history];
```

- 启用时：`recentFullTextCount` 取代 `historyLimit` 控制「原文楼数」；旧楼层变成 `<过往记忆>` 不再丢。
- `deep` 可填：`miscStore` 的全部 `largeSummaries.join('\n')`（世界级阶段记忆），或留空。世界书匹配 `matchCtx` 仍用最近原文即可。

### 步骤 ⑥（可选）UI
- 在某楼层旁显示「已总结」小标记；或在调试「查看发送的 Prompt」(`promptSent`) 里能看到 `<过往记忆>` 块（已自动包含，便于核对）。

## 4. 与已有「总结 / 生平压缩」的关系

| 模块 | 对象 | 作用 |
|---|---|---|
| **本指导：叙事记忆** | 正文楼层 | 把旧楼层换成小/大总结塞回**正文生成**上下文（解决"记不住前文"）|
| 杂项演化·总结 | 世界/剧情线 | 产出 small/large 总结（**数据来源**，喂给本模块）|
| 生平压缩(memoryStore) | 逐角色 | 压缩单个角色的 short/long 记忆 |

三者互补：杂项总结**生产**摘要 → 叙事记忆**消费**摘要注入正文；生平压缩管角色档，可作 `[深度记忆档案]` 备选来源。

## 5.5 实际实现（已落地，关键词召回版）

最终实现的是**关键词召回**（对齐源前端「未配置向量时用标题关键词模式」），而非单纯按楼层远近的三档注入：

- `settingsStore.narrativeMemory{enabled,recentFullTextCount,distantKeywordThreshold,recallTopK,recallMinScore,requestTimeout}`（默认关；最近5/远层阈值200/TopK6/超时90）。
- `systems/narrativeMemory.ts`：`tokenize`（中文 2-gram + 拉丁词）+ `recallFacts(query, facts, topK, minScore)`（关键词命中度排序 Top-K）+ `buildNarrativeHistory`（最近 N 楼原文 + 召回事实拼 `<相关记忆>` system 块；超 `distantKeywordThreshold` 名次只留标题）。
- **召回源**：`miscStore` 的 `smallSummaries / largeSummaries / worldEvents`（杂项演化产出）。**查询**=当前输入 + 最近一条 assistant 正文。
- `App.tsx` `callApi`：启用时构造 facts + query → `buildNarrativeHistory` → 注入 `<相关记忆>`；关闭时回退 `historyLimit` 切片。
- **UI**：
  - **设置界面主页**新增 **「🧠 叙事记忆」入口**（独立页 `narrative-memory`，不在变量管理里）：三张说明卡 + 标题关键词模式提示 + 启用开关 / 最近全文条数 / Top-K / 最低命中分 / 远层阈值 / 请求超时。
  - 右侧游戏导航 **「🧠 记忆」按钮** → `SummaryPanel` 看小/大总结。
- 纯本地、零额外 API 调用（关键词模式）。默认 `enabled:false`。

**LLM 两步法（可选，解决"只召回最新"）**——对齐源前端「发送前整理 + 回复后写入」：
- `narrativeMemory` 加 `llmMode / compileModelId / ingestModelId`；`settingsStore` 加独立 `nmApi`（可与正文共用）。
- **发送前整理**（`narrativeCompile`）：把当前情境喂给 LLM，产出**检索关键词**（`NM_COMPILE_PROMPT`）→ 用它做关键词召回，找"相关"而非"最新"。在 `callApi` 里 await（阻塞、轻量）。
- **回复后写入**（`runNarrativeIngestPhase`）：每轮正文喂给 LLM 抽取**长期事实**（`NM_INGEST_PROMPT`，输出 `{facts:[{title,text,keywords}]}`）→ 存入 `miscStore.narrativeFacts`（上限 300）。接入 `runPostNarrativePhases`，独立并发。
- 召回源优先 `narrativeFacts`，再叠加小/大总结 + 世界大事。
- UI：设置→🧠 叙事记忆→「使用的模型（LLM 两步法）」：启用开关 + 共用/独立 API + 发送前整理模型 + 回复后写入模型（留空用默认）。
- 「🧠 记忆」面板新增 **「长期事实」tab**（看抽取结果，可删）。
- 后续可加 embedding 档（similarity/hybrid）升级为向量召回，接口已按 facts/query 抽象好。

## 6. 注意事项

- **回退链很重要**：small 缺则用原文、large 缺则用小总结/原文（`BL` 同款），保证没总结的楼层也不会变空。
- **只压历史，不压当前输入**：本轮 user 输入和最近 `recentFullTextCount` 楼始终原文。
- **`<过往记忆>` 放最前、role=system**：和 fanren-remake 一致，模型把它当背景而非对话。
- **token 预算**：`recentFullTextCount` 越大越贵；小总结窗口越大记得越细但也越长。默认 6+20 是个均衡起点。
- 关闭时行为与现状完全一致（只留最近若干楼），可安全灰度。
