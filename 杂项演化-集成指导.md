# 杂项演化（misc evolution）集成指导

把 **fanren-remake** 的「杂项演化」移植到 **zhushen-space**。本文档说明：① 杂项演化到底管什么；② 在哪找原始实现；③ 完整的输出契约；④ zhushen-space 落地方案（store / AI 阶段 / 解析 / UI / 预设）。

---

## 0. 一句话

杂项演化是**第 4 个并发 AI 演化阶段**（与 物品 / 主角 / NPC 演化并列），专门维护「不属于任何角色」的世界级杂项状态：**分段记忆总结、天气、世界大事、主角任务、小地图/区域结构**。它**只读正文、只写变量**，绝不碰角色本体/物品/技能。

---

## 1. 杂项演化管什么（5 大块）

| 模块 | 作用 | 指令 |
|---|---|---|
| ① 分段记忆总结 | 每轮滚动记录剧情摘要（小总结 120-220 字 / 大总结 90-180 字）| `addSmallSummary("…")` / `addLargeSummary("…")` |
| ② 天气 | 维护状态栏天气 | `timeLocation.weather = "小雨"` |
| ③ 世界大事 | 区域格局/势力/资源/声望级的大事件（宁缺毋滥）| `addWorldEvent / updateWorldEvent / deleteWorldEvent` |
| ④ 主角任务 | 仅 `T_<数字>` 任务的新增/更新/删除（不在此判完成/失败）| `set(...)` / `add("T_x", {...})` / `de("T_x")` |
| ⑤ 小地图 + 区域 | ASCII 字符画地形 + 百分比坐标的元素图层；区域结构 | `add("SCENE_MAP", {...})` / `createLocation / updateRegion / deleteLocation` |

**绝对禁令**（杂项阶段不许做）：改角色本体表(B1/C*/G*/P*)、物品/装备/货币、技能/词条/属性、非 T 开头的 de、改 `timeLocation.currentTime/currentLocation/season`。

## 3. 输出契约（指令清单，原样照搬）

AI 只输出一个 `<upstore>` 块，**一行一条指令**，无变化则 `<upstore></upstore>`：

```
<upstore>
addSmallSummary("宿主在港口区完成与情报中间人的接触，对方确认仓库夜间会有秘密交接……")
addLargeSummary("港口渗透线进入潜伏阶段：宿主已建立情报接头渠道，下一阶段核心目标为验证仓库线索……")
timeLocation.weather = "小雨"
addWorldEvent("0001年 03月 16日", "碧蓝浩瀚·港口城", "黑市地下网络遭大规模清查，三处据点同时被封锁……")
set({"0":"T_17","1":"潜入南巷仓库","2":"在不惊动守卫前提下确认仓库货单与接头暗号","3":"获得黑市核心情报","4":"触发通缉并失去黑市渠道","5":"进行中/三阶中期","startTime":"","endTime":"0001年05月18日 23:30"})
add("T_17", {"2":"已摸清外墙巡逻节奏，下一步需核对内库暗号","5":"进行中/三阶中期"})
de("T_3")
add("SCENE_MAP", {"1":{"size":[25,25],"terrain_patches":[{"tile":".","rect":[0,0,24,24]},{"tile":"~","rect":[6,8,17,10]},{"tile":"#","rect":[10,12,14,15]}],"elements":[{"coords":[36.0,44.0],"type":"player","name":"你","icon":"fa-user"},{"coords":[60.0,44.0],"type":"npc","name":"黑市接头人","icon":"fa-user-secret"}]}})
</upstore>
```

### 关键字段规范
- **任务 ID**：固定 `T_<数字>` 不补零（`T_15` ✓；`T15`/`T_015` ✗）。新建用 `${next_available_task_id}`；更新必须用任务列表里的精确 ID；任务名不是 ID。
- **任务字段**：`0`=ID、`1`=名、`2`=描述、`3`=成功奖励、`4`=失败惩罚、`5`=`进行中/阶位`；`startTime`/`endTime` 用绝对时间 `0001年06月01日 09:30`，无则空串。完成/失败**不在杂项阶段判**。
- **天气**：`timeLocation.weather = "…"` 独占一行；短词（晴/阴/小雨/山雨欲来）。
- **世界大事**：只记区域格局/势力/资源/声望级；普通对话/个人事不记；无则不输出。
- **SCENE_MAP**：`terrain` 等宽矩形字符画，或 `terrain_patches`（`{tile,rect:[x1,y1,x2,y2]}` 增量补丁，推荐）；`elements` 坐标 `[0,100]` 百分比；默认 25×25，1 格=1 米；必须覆盖全部在场角色，移除离场幽灵元素。

## 5. 数据结构设计（新 store `miscStore.ts`，key `drpg-misc`）

```ts
export interface MiscTask {
  id: string;            // "T_17"
  name: string;          // 1
  desc: string;          // 2
  reward: string;        // 3 成功奖励
  penalty: string;       // 4 失败惩罚
  status: string;        // 5 "进行中/三阶中期"
  startTime: string;
  endTime: string;
  addedAt: number;
}
export interface WorldEvent {
  id: string;            // 自动生成 "W_<n>"
  time: string;
  location: string;
  desc: string;
}
export interface SceneMap {
  size: [number, number];
  terrain?: string[];                 // 全量字符画（每行一个等宽字符串）
  terrainPatches?: { tile: string; rect: [number, number, number, number] }[];
  elements: { coords: [number, number]; type: 'player'|'npc'|'object'|'scenery'|'poi'; name: string; icon?: string; size?: string }[];
}
export interface MiscRegion { id: string; name: string; desc?: string; parent?: string }

interface MiscState {
  tasks: MiscTask[];
  worldEvents: WorldEvent[];
  smallSummaries: string[];   // 滚动保留最近 N 条
  largeSummaries: string[];
  weather: string;
  sceneMap: SceneMap | null;
  regions: MiscRegion[];
  // 设置（独立开关 + 提示词 + API，沿用 memoryStore/playerStore 模式）
  settings: { enabled: boolean; prompt: string; smallKeep: number; largeKeep: number };
  miscApi: ApiConfig; miscUseSharedApi: boolean; /* + models 三件套 */
  // actions
  upsertTask: (t: MiscTask) => void; updateTask: (id: string, patch: Partial<MiscTask>) => void; removeTask: (id: string) => void;
  addWorldEvent: (e: Omit<WorldEvent,'id'>) => void; updateWorldEvent: (id: string, patch: Partial<WorldEvent>) => void; removeWorldEvent: (id: string) => void;
  pushSmall: (s: string) => void; pushLarge: (s: string) => void;
  setWeather: (w: string) => void; setSceneMap: (m: SceneMap) => void;
  upsertRegion: (r: MiscRegion) => void; removeRegion: (id: string) => void;
  nextTaskId: () => string;  // 扫描 tasks 取最大 T_<n>+1
}
```

> `${next_available_task_id}` 由 `nextTaskId()` 计算后注入提示词。`${current_tasks}` / `${current_scene_map}` / `${world_events}` 由 store 序列化注入。

async function runMiscEvolutionPhase(narrative: string) {
  const { settings, miscApi, miscUseSharedApi } = useMisc.getState();
  if (!settings.enabled) return;
  const ss = useSettings.getState();
  const api = miscUseSharedApi ? (ss.textUseSharedApi ? ss.api : ss.textApi) : miscApi;
  if (!api.baseUrl || !api.apiKey) return;

  // 注入上下文：把 ${...} 占位符替换为实时 store 数据
  const m = useMisc.getState();

    .replace('${story_text}', trimNarrative(narrative))
    .replace('${current_tasks}', serializeTasks(m.tasks))
    .replace('${current_scene_map}', JSON.stringify(m.sceneMap ?? {}))
    .replace('${world_events}', serializeEvents(m.worldEvents))
    .replace('${weather}', m.weather)
    .replace('${next_available_task_id}', m.nextTaskId())
    /* …其余占位符… */;

## 7. 指令解析（`src/systems/miscParser.ts` 或并入 stateParser）

从 `<upstore>` 块逐行解析。注意 `add`/`set`/`de` 与现有物品/NPC 指令冲突——杂项的 `add`/`set` 只针对 `T_<数字>` 与 `SCENE_MAP`，用 ID 前缀区分路由：

```ts
export function applyMiscCommands(reply: string) {
  const block = (reply.match(/<upstore>([\s\S]*?)<\/upstore>/i)?.[1] ?? '');
  const M = useMisc.getState();
  for (const line of block.split('\n').map(s => s.trim()).filter(Boolean)) {
    let m: RegExpExecArray | null;
    if ((m = /^addSmallSummary\("([\s\S]*)"\)$/.exec(line))) { M.pushSmall(unquote(m[1])); continue; }
    if ((m = /^addLargeSummary\("([\s\S]*)"\)$/.exec(line))) { M.pushLarge(unquote(m[1])); continue; }
    if ((m = /^timeLocation\.weather\s*=\s*"([^"]*)"$/.exec(line))) { M.setWeather(m[1]); continue; }
    if ((m = /^addWorldEvent\("([^"]*)",\s*"([^"]*)",\s*"([\s\S]*)"\)$/.exec(line))) { M.addWorldEvent({ time:m[1], location:m[2], desc:unquote(m[3]) }); continue; }
    if ((m = /^updateWorldEvent\("([^"]+)",\s*"([^"]*)",\s*"([^"]*)",\s*"([\s\S]*)"\)$/.exec(line))) { M.updateWorldEvent(m[1], { time:m[2], location:m[3], desc:unquote(m[4]) }); continue; }
    if ((m = /^deleteWorldEvent\("([^"]+)"\)$/.exec(line))) { M.removeWorldEvent(m[1]); continue; }
    if ((m = /^de\("(T_\d+)"\)$/.exec(line))) { M.removeTask(m[1]); continue; }
    if ((m = /^set\((\{[\s\S]*\})\)$/.exec(line))) { const o = safeJson(m[1]); if (o?.['0']?.startsWith?.('T_')) M.upsertTask(taskFromCols(o)); continue; }
    if ((m = /^add\("(T_\d+)",\s*(\{[\s\S]*\})\)$/.exec(line))) { M.updateTask(m[1], patchFromCols(safeJson(m[2]))); continue; }
    if ((m = /^add\("SCENE_MAP",\s*(\{[\s\S]*\})\)$/.exec(line))) { const o = safeJson(m[1]); if (o?.['1']) M.setSceneMap(normalizeSceneMap(o['1'])); continue; }
    if ((m = /^createLocation\(([\s\S]*)\)$/.exec(line))) { /* upsertRegion */ continue; }
    if ((m = /^updateRegion\(([\s\S]*)\)$/.exec(line)))   { /* upsertRegion */ continue; }
    if ((m = /^deleteLocation\("([^"]+)"\)$/.exec(line))) { /* removeRegion */ continue; }
  }
}
```

> `taskFromCols` 把 `{"0":..,"1":..}` 列号映射到 `MiscTask` 字段；`normalizeSceneMap` 把 `terrain`/`terrain_patches` 统一成 `SceneMap`。SCENE_MAP 的 `add` 要在物品/NPC 的 `add` 解析器**之前**用 `"SCENE_MAP"`/`"T_…"` 字面量短路，避免被当成物品指令。

## 8. UI

新增右侧导航按钮，或单独面板（参考 `NpcPanel`/`CharacterPanel` 弹窗模式）：

- **📋 任务**：`tasks` 列表卡片（名/描述/奖励/惩罚/状态/起止时间），手动可删。
- **📰 世界大事**：`worldEvents` 时间线（time @ location：desc）。
- **🌤 天气**：顶部状态栏把硬编码文字换成 `useMisc(s=>s.weather)`（顺带把日期/地点也变量化）。
- **📜 剧情总结**：small/large 两段滚动列表（可与「生平压缩」入口并列）。
- **🗺 小地图**（核心，新组件 `MiniMap.tsx`）：
  - 把 `terrain`/`terrain_patches` 合成成 `size[0]×size[1]` 的字符网格，用 `font-mono` 渲染（每字符一个等宽格）。
  - `elements` 按 `coords` 百分比绝对定位浮在网格上，`icon` 用 FontAwesome（或自带 emoji 映射），`size` 控制图标大小，`name` 作 tooltip/标签。
  - 地形字符 → 颜色映射（`.`=地面、`~`=水、`^`=山/边界、`#`=建筑…），可做一个 `TILE_CLS` 表。

```tsx
// MiniMap 渲染思路
function MiniMap({ map }: { map: SceneMap }) {
  const [w, h] = map.size;
  const grid = buildGrid(map);            // terrain + patches → string[h]，每行 w 字符
  return (
    <div className="relative font-mono leading-none text-[10px] bg-void">
      {grid.map((row, y) => <div key={y} className="whitespace-pre">{[...row].map((ch,x)=>
        <span key={x} className={TILE_CLS[ch] ?? 'text-dim/40'}>{ch}</span>)}</div>)}
      {map.elements.map((e,i) => (
        <div key={i} title={e.name}
          className="absolute -translate-x-1/2 -translate-y-1/2"
          style={{ left:`${e.coords[0]}%`, top:`${e.coords[1]}%` }}>
          <i className={`fa ${e.icon ?? 'fa-circle'} ${ELEM_CLS[e.type]}`} />
        </div>
      ))}
    </div>
  );
}
```

## 10.5 实际实现（已落地）

已按本指导实现，**小地图（SCENE_MAP / 区域）暂未做**，并为轮回乐园加了**双时间体系**：

- **store** `miscStore.ts`（key `drpg-misc`）：tasks / worldEvents / smallSummaries / largeSummaries / weather / **paradiseTime（轮回力时间）** / **worldTime（任务世界时间）** / worldName + 设置(enabled/prompt/keep) + 独立 API。
- **双时间**：
  - `paradiseTime` 轮回力时间「轮回力X年X月X日 X时X分」（主神空间日历）
  - `worldTime` 角色当前任务世界的具体时间（如进入二战战役 → 1943年02月02日 14:30），`worldName` 为世界/场景名
  - AI 指令：`timeLocation.paradiseTime/worldTime/worldName/weather = "…"`，由杂项演化阶段**负责推进**（原版禁止改时间的条款已为本项目放开）。
- **解析** `systems/miscParser.ts`：`applyMiscCommands` 解析 `<upstore>` 内 总结/双时间/天气/世界大事/`T_` 任务；用 `T_`/字面量前缀短路，**不碰**物品/NPC 的 add/set/de。
- **阶段** `App.tsx` `runMiscEvolutionPhase` 并入 `runPostNarrativePhases`，独立并发；用自己的 API；注入 `${current_paradise_time}/${current_world_time}/${current_world_name}/${weather}/${current_tasks}/${world_events}/${next_available_task_id}/${story_text}`。
- **入口/UI**：
  - 设置 → 变量管理 → **「🧩 杂项演化」按钮** → 独立 `MiscManager`（演化设置 / API 两个 tab；提示词即预设，可直接编辑 + 恢复默认；可手动初始化双时间）。
  - 右侧导航 **「📋 任务」** → `MiscPanel`（顶部双时间+天气，三 tab：任务 / 世界大事 / 剧情总结）。
  - 顶部状态栏时间/地点/天气改为读 `miscStore`（轮回力时间 + 世界·世界时间·天气）。
- **预设（条目模型 + 导入导出，完整 14 条）**：与主角/NPC 演化一致——`miscStore` 存 `entries: MiscPresetEntry[]`。内置默认 = **原版 `prompts.misc_management` 全部 13 条规则 + 1 条「双时间体系」**（共 14 条：双时间 / 身份与上下文 / 在场白名单 / 范围禁令 / 指令定义 / Standalone任务奖励·质量·阶位对齐·仅建任务 / 任务与小地图细则 / 时间地点与世界大事 / COT流程 / 易错点审查 / 输出格式），从 `src/data/miscDefaultPreset.json` 载入。`buildMiscSystemPrompt` 拼启用条目，运行时替换全部 `${...}`（含原版 `current_time/current_location/world_map_pois/current_scene_map/player_name` 等，无小地图者置「未启用」）。`MiscManager` 设置页有 **导入 JSON / 导出 / 恢复默认** 按钮 + 逐条开关/编辑。标准预设文件 **`预设/杂项演化.json`**（UTF-8 无 BOM，14 条）与 `src/data/miscDefaultPreset.json` 同源。小地图相关规则保留为可关闭条目（渲染未实现，AI 若输出 SCENE_MAP 会被解析器忽略）。
- 默认 `enabled:false`。

## 11. 注意事项

- **`add`/`set`/`de` 命名冲突**：杂项的 `add("T_…")`/`add("SCENE_MAP")`/`set({"0":"T_…"})`/`de("T_…")` 必须用字面量前缀短路，**早于**物品(`createItem`)、NPC(`add("C1",…)`) 的解析，避免互相误吞。建议杂项解析器只认 `T_`/`SCENE_MAP` 开头，其余忽略。
- **任务完成/失败不在杂项阶段判**：那是主线/结算的事；杂项只新增/更新/删除任务。
- **世界大事宁缺毋滥**：阈值很高，多数回合应 0 条。
- **小地图增量优先**：优先 `terrain_patches` + 只更 `elements`，避免每轮全量重绘；必须清理离场「幽灵元素」。
- **与「生平压缩」区分**：杂项的 small/large 是**世界/剧情级**滚动摘要；`memoryStore` 是**逐角色**记忆压缩。两者可共存，UI 入口建议分开。
- 默认 `enabled:false`，避免无谓 API 调用（与原版一致）。
